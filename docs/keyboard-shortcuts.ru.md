# Горячие клавиши и советы {#keyboard-shortcuts}

На этой странице перечислены сочетания клавиш и советы по эффективной работе с Lingtrain Aligner.

## Сочетания клавиш браузера {#browser-shortcuts}

Поскольку Lingtrain — веб-приложение, стандартные сочетания клавиш браузера работают во всём интерфейсе:

| Сочетание | Действие |
|-----------|----------|
| `Ctrl+R` / `Cmd+R` | Перезагрузить страницу (полезно, если интерфейс завис) |
| `Ctrl+F` / `Cmd+F` | Поиск текста на текущей странице |
| `Ctrl+0` / `Cmd+0` | Сбросить масштаб до 100% |
| `Ctrl++` / `Cmd++` | Увеличить масштаб |
| `Ctrl+-` / `Cmd+-` | Уменьшить масштаб |
| `Ctrl+Tab` | Переключиться на следующую вкладку браузера |
| `Ctrl+Shift+Tab` | Переключиться на предыдущую вкладку |
| `Alt+Left` / `Cmd+Left` | Назад в истории браузера |
| `F11` | Переключить полноэкранный режим (больше места для редактора) |

## Навигация по приложению {#app-navigation}

### Навигация по вкладкам {#tab-navigation}

Алайнер имеет три основные вкладки: **Documents** (Документы), **Alignments** (Выравнивания) и **Create** (Создание). Нажимайте на названия вкладок для переключения. Активная вкладка выделена.

**Совет:** Используйте вкладку Documents для подготовки текстов, Alignments для обработки и Create для экспорта. Работайте с ними последовательно для естественного рабочего процесса.

### Боковая панель {#sidebar-navigation}

Левая боковая панель обеспечивает навигацию к различным разделам приложения. Нажмите на значок алайнера, чтобы перейти в рабочее пространство.

### Выбор языка {#language-selector}

На вкладке Documents выпадающие списки языков вверху позволяют быстро переключаться между контекстами исходного и целевого языков. Выбранные языки сохраняются между сессиями.

## Советы по работе с редактором {#editor-tips}

### Эффективная навигация по страницам {#page-navigation}

- Используйте **поле ввода номера страницы** (между кнопками «Prev» и «Next») для прямого перехода на нужную страницу
- Настройте **количество строк на странице** (10, 20 или 50) под свои задачи: 10 — для детального просмотра, 50 — для быстрого сканирования
- Редактор сохраняет настройку количества строк в локальном хранилище браузера

### Работа с парами выравнивания {#working-with-pairs}

- **Наведите курсор на ячейку** для отображения кнопок действий (редактировать, кандидаты, удалить, добавить строку)
- **Нажмите кнопку редактирования** на ячейке для изменения текста — нажмите Enter или кликните вне поля для сохранения
- **Используйте браузер кандидатов** для поиска правильного предложения-соответствия, когда пара выровнена неверно
- **Листайте кандидатов** стрелками Previous/Next для просмотра соседних предложений из исходного текста
- **Добавляйте пустые строки** выше или ниже для создания пространства для ручных корректировок

### Навигация по визуализации батчей {#visualization-navigation}

- Каждая карточка визуализации имеет кнопку **«Open in editor»**, которая прокручивает редактор к соответствующему батчу
- Нажимайте на карточки батчей в секции визуализации для проверки конкретных батчей
- Показатели качества на каждой карточке дают мгновенное представление о том, какие батчи требуют внимания

### Навигация по конфликтам {#conflict-navigation}

- Используйте кнопки **Previous/Next** в разделе конфликтов для пошагового просмотра
- Счётчик конфликтов (например, «3 / 31») показывает вашу позицию в списке
- Нажмите **«Open in editor»** на любом конфликте для перехода к его местоположению в редакторе
- Включайте флажки **«Handle start»** и **«Handle finish»** только при необходимости обработки краевых конфликтов

## Советы для опытных пользователей {#power-user-tips}

### Оптимальный порядок работы {#workflow-order}

Для максимально быстрого выравнивания:

1. **Подготовьте тексты офлайн** — очистите, отформатируйте и добавьте теги разметки в текстовом редакторе перед загрузкой
2. **Загрузите оба текста** и убедитесь, что число предложений в ожидаемом диапазоне
3. **Проверьте панель Marks** — убедитесь в совпадении количества тегов между текстами
4. **Выровняйте один батч** — проверьте визуализацию, прежде чем обрабатывать остальные
5. **Скорректируйте сдвиг при необходимости** — на основе диагонали первого батча
6. **Обработайте оставшиеся батчи** группами по 3-5
7. **Разрешите конфликты один раз** после обработки всех батчей
8. **Просматривайте в редакторе** только оставшиеся конфликты — не проверяйте каждую пару вручную
9. **Экспортируйте** нужные форматы

### Подготовка текстов: приёмы {#text-preparation}

Используйте эти приёмы в текстовом редакторе перед загрузкой:

- **Поиск и замена** для массового добавления тегов разметки (например, найти заголовки глав по шаблону и добавить `%%%%%h1.`)
- **Регулярные выражения** для нормализации пробелов, исправления пунктуации или удаления номеров страниц
- **Инструменты сравнения (diff)** для проверки структурного соответствия исходного и целевого текстов

### Эффективное разрешение конфликтов {#efficient-resolution}

- Всегда запускайте **автоматическое разрешение первым** — оно обрабатывает 70-90% конфликтов без ручного вмешательства
- После автоматического разрешения просматривайте оставшиеся конфликты, начиная с **самых маленьких** — они обычно решаются быстро
- Для крупных конфликтов (много предложений) сначала проверьте, действительно ли тексты соответствуют друг другу в данном фрагменте
- Используйте **браузер кандидатов** вместо свободного редактирования текста, когда это возможно

### Хранение настроек {#local-storage}

Lingtrain сохраняет в локальном хранилище браузера:

- Выбранные языки (вкладка Documents)
- Настройки количества строк на странице (редактор)
- Состояния панелей интерфейса (развёрнуты/свёрнуты)

Эти настройки сохраняются между сессиями, поэтому их не нужно перенастраивать каждый раз.

### Несколько вкладок браузера {#multiple-tabs}

Вы можете открыть несколько выравниваний в отдельных вкладках браузера для сравнения или одновременной работы над несколькими проектами. Каждая вкладка поддерживает собственное состояние независимо.

### Использование поиска браузера для больших текстов {#browser-find}

При работе с большими выравниваниями в редакторе:

1. Установите 50 строк на страницу для максимального видимого содержимого
2. Используйте `Ctrl+F` / `Cmd+F` для поиска текста на текущей странице
3. Переходите между страницами для поиска текста на других страницах

Примечание: поиск браузера работает только на текущей отрисованной странице, а не по всему выравниванию.

### Советы по экспорту {#export-tips}

- **Предварительный просмотр** — используйте кнопку «Generate preview» на вкладке Create для проверки корректности вывода
- **Попробуйте разные направления абзацев** — переключение между абзацной структурой исходного и целевого текста может значительно улучшить читаемость
- **Протестируйте разные стили** — три стиля подсветки подходят для разных задач (Simple для печати, Pastel Fill для чтения с экрана)
- **Скачивайте файл .lt** периодически как резервную копию — это единственный формат, сохраняющий полное состояние выравнивания для повторного импорта
