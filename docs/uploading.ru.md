# Загрузка текстов {#uploading}

Вкладка **Тексты** предназначена для загрузки, предпросмотра и управления исходными текстами перед выравниванием.

## Подготовка текстов {#preparation}

Перед загрузкой подготовьте ваши текстовые файлы:

1. **Используйте формат plain text** (`.txt`). Система обрабатывает только обычный текст, а не PDF, DOCX или другие форматы.
2. **Удалите лишнее содержимое** — номера страниц, сноски, примечания издательства, оглавление, предисловие переводчика (если вы не хотите их выравнивать).
3. **Разделяйте абзацы пустыми строками**. Разбиватель использует границы абзацев для улучшения определения предложений.
4. **Добавьте теги разметки** (см. ниже) для заголовков, имён авторов и других метаданных.

## Выбор языков {#languages}

В верхней части страницы «Тексты» выберите **исходный** («Из») и **целевой** («В») языки с помощью выпадающих меню. Этот выбор определяет:

- Какие языко-специфичные правила разбиения на предложения применяются
- Как классифицируются загруженные документы
- Какие флаги отображаются

![Страница документов — пустое состояние](img/documents-empty.en.png)

## Загрузка файлов {#upload}

Каждая языковая панель имеет область загрузки с поддержкой перетаскивания. Вы можете:

- **Перетащить** файл `.txt` в область загрузки
- **Кликнуть** по области загрузки для открытия выбора файла

Флажок **Очистить текст** применяет базовую нормализацию (удаление лишних пробелов, исправление кодировки) при загрузке. Включите его, если ваш текст содержит артефакты форматирования.

После загрузки документ появляется в списке под областью загрузки, а панель **Предпросмотр предложений** обновляется, показывая результат разбиения.

![Страница документов с загруженными текстами](img/documents-uploaded.en.png)

## Теги разметки {#markup}

Lingtrain использует специальные теги разметки для отделения метаданных (заголовков, цитат) от основного текста. Теги размещаются **в конце строки** с префиксом `%%%%%`.

### Поддерживаемые теги {#tags}

| Тег | Назначение | Пример |
|-----|-----------|--------|
| `%%%%%author.` | Имя автора | `Кристи Агата%%%%%author.` |
| `%%%%%title.` | Название книги | `Синяя герань%%%%%title.` |
| `%%%%%h1.` | Заголовок 1-го уровня | `Глава первая%%%%%h1.` |
| `%%%%%h2.` | Заголовок 2-го уровня | `Часть I%%%%%h2.` |
| `%%%%%h3.` | Заголовок 3-го уровня | `Раздел 1.1%%%%%h3.` |
| `%%%%%h4.` | Заголовок 4-го уровня | `Подраздел%%%%%h4.` |
| `%%%%%h5.` | Заголовок 5-го уровня | `Малый заголовок%%%%%h5.` |
| `%%%%%qtext.` | Текст цитаты | `Быть или не быть%%%%%qtext.` |
| `%%%%%qname.` | Атрибуция цитаты | `Уильям Шекспир%%%%%qname.` |
| `%%%%%image.` | Ссылка на изображение | `illustration.jpg%%%%%image.` |
| `%%%%%translator.` | Упоминание переводчика | `Перевод Иванова И.И.%%%%%translator.` |
| `%%%%%divider.` | Разделитель секций | `* * *%%%%%divider.` |

### Как работают теги {#how-tags-work}

- Строки с тегами **исключаются из алгоритма выравнивания** — они не будут сопоставляться с предложениями другого языка.
- Теги **сохраняются как метаданные** в базе выравнивания и используются для форматирования при экспорте (титульные страницы, заголовки глав, оформление цитат).
- Теги `author` и `title` — самые важные: они формируют титульную страницу в экспортированной HTML-книге.
- **Оба текста должны иметь одинаковую разметку**. Например, если в английском тексте есть теги `%%%%%author.` и `%%%%%title.`, в русском тексте они тоже должны быть.

### Пример {#markup-example}

**Файл английского текста (`agata_geran_en.txt`):**

```
Agatha Christie%%%%%author.
The Blue Geranium%%%%%title.


'When I was down here last year -' said Sir Henry Clithering, and stopped.

His hostess, Mrs Bantry, looked at him curiously. The Ex-Commissioner
of Scotland Yard was staying with old friends of his, Colonel and Mrs Bantry,
who lived near St Mary Mead.
```

**Файл русского текста (`agata_geran_ru.txt`):**

```
Кристи Агата%%%%%author.
Синяя герань%%%%%title.


— Когда я был здесь в прошлом году… — начал сэр Генри Клитеринг и умолк.

Хозяйка дома, миссис Бантри, посмотрела на него с любопытством.
Экс-комиссар Скотленд-Ярда, их старый приятель, гостил у них с мужем
в их имении, расположенном неподалеку от Сент-Мэри-Мид.
```

## Предпросмотр предложений {#preview}

После загрузки панель **Предпросмотр предложений** показывает, как каждый документ был разбит на отдельные предложения. Каждая строка пронумерована и представляет одно предложение, с которым будет работать алгоритм выравнивания.

Строки с тегами разметки отображают тип тега в виде цветного значка (например, `author`, `title`, `paragraph`).

Значок `paragraph` отмечает последнее предложение в абзаце — эта информация используется позже для восстановления структуры абзацев в экспортированной книге.

![Предпросмотр предложений с метками](img/marks-panel.en.png)

Можно настроить количество строк на странице (10, 20 или 50) и переключаться между страницами. Используйте кнопку **Перейти к странице** для перехода к конкретной странице или кнопку **скачивания** для сохранения разбитого текста в файл.

## Панель меток {#marks}

Панель **Метки** внизу показывает все теги разметки, найденные в обоих документах рядом. Это ваш этап проверки — убедитесь, что:

- Оба документа имеют **одинаковое количество меток** для каждого типа тега
- Содержимое меток совпадает (например, имя автора присутствует на обоих языках)

Несовпадение количества меток может вызвать проблемы с выравниванием, поскольку метки определяют структурные границы, на которые опирается алгоритм.

## Прокси-документы {#proxy}

Для языковых пар, где модель эмбеддингов имеет ограниченные обучающие данные (например, башкирский, чувашский или другие малоресурсные языки), можно использовать **прокси-документы** для улучшения качества выравнивания.

Прокси-документ — это **машинный перевод** разбитого текста, который служит посредником. Порядок действий:

1. Загрузите и разбейте исходный текст обычным образом
2. **Скачайте разбитый текст** с помощью кнопки скачивания в предпросмотре предложений
3. **Переведите его** с помощью любого сервиса машинного перевода (Google Translate, DeepL и т.д.)
4. **Загрузите перевод обратно** как прокси-документ в разделе «Прокси-документы» на странице деталей выравнивания

Прокси должен содержать **точно такое же количество строк**, как и оригинальный разбитый текст. Алгоритм выравнивания использует прокси для сопоставления, а оригинальный текст сохраняется в финальном результате.

Текст прокси отображается в виде подстрочных аннотаций в редакторе, помогая проверять качество выравнивания, даже если вы не владеете одним из языков.
