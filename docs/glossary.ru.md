# Глоссарий {#glossary}

Данный глоссарий содержит определения ключевых терминов и понятий, используемых в Lingtrain Aligner и его документации.

---

### Агрегация (Aggregation) {#aggregation}

Процесс объединения нескольких эмбеддингов предложений в единый вектор при разрешении конфликтов. Lingtrain поддерживает несколько методов агрегации: взвешенное среднее (по умолчанию), масштабирование по длине, max pooling и логарифмическое масштабирование. Метод агрегации определяет, каким образом векторы отдельных предложений объединяются при группировке предложений в ходе разрешения конфликтов.

### База данных выравнивания (Alignment Database) {#alignment-database}

SQLite-файл (расширение `.lt`), в котором хранятся все данные выравнивания для одного проекта. Содержит разбитые на предложения тексты обоих языков, вычисленные эмбеддинги, индекс документа (карту соответствия предложений), метаданные батчей, историю правок и теги разметки. Этот файл можно экспортировать для резервного копирования и повторного импорта.

### Батч (Batch) {#batch}

Сегмент исходного текста, обрабатываемый как единое целое в процессе выравнивания. По умолчанию каждый батч содержит 200 предложений исходного текста и пропорционально соответствующие предложения целевого текста (плюс перекрытие «окна»). Большие тексты делятся на несколько батчей для поэтапной обработки, отслеживания прогресса и контроля использования памяти.

### Визуализация (Visualization) {#visualization}

Точечный график, генерируемый для каждого обработанного батча. На одной оси показаны позиции предложений исходного текста, на другой — позиции соответствующих целевых предложений. Чистая диагональ указывает на хорошее выравнивание; разбросанные точки сигнализируют о проблемах. Визуализация помогает определить батчи, требующие коррекции параметров или ручной проверки.

### Выравнивание (Alignment) {#alignment}

Основной процесс сопоставления предложений исходного и целевого текстов. В Lingtrain выравнивание — это проект, содержащий все данные, связанные с установлением соответствий между двумя текстами: разбитые тексты, вычисленные эмбеддинги, итоговую карту пар предложений, историю разрешения конфликтов и ручные правки.

### GUID {#guid}

Globally Unique Identifier (глобально уникальный идентификатор). Случайная строка, однозначно идентифицирующая каждое выравнивание и документ в Lingtrain. GUID используется в URL-адресах, эндпоинтах API и путях к файлам для обращения к конкретным ресурсам.

### Документ (Document) {#document}

Текстовый файл, загруженный в Lingtrain для выравнивания. Документы — это текстовые файлы (`.txt`), привязанные к конкретному языку. После загрузки текст разбивается на предложения с использованием языковых правил и сохраняется в системе для использования в выравниваниях.

### Индекс документа (Document Index) {#document-index}

Внутренняя структура данных, фиксирующая соответствие между исходными и целевыми предложениями. Организована по батчам и содержит пары `(source_line_ids, target_line_ids)` для каждой выровненной строки. Индекс документа — основной результат процесса выравнивания.

### Инференс (Inference Type) {#inference-type}

Способ вычисления эмбеддингов: «local» (на собственном оборудовании сервера) или «api» (через внешние API, такие как Hugging Face или OpenAI). Локальный инференс обеспечивает меньшую задержку и отсутствие попозапросных расходов, но требует GPU-ресурсов. API-инференс предоставляет доступ к дополнительным моделям.

### CAT-инструмент (CAT Tool) {#cat-tool}

Инструмент автоматизированного перевода (Computer-Assisted Translation). Программное обеспечение для профессиональных переводчиков, использующее переводческую память, глоссарии и другие средства. Примеры: SDL Trados Studio, memoQ, OmegaT, Memsource, Wordfast. Lingtrain экспортирует TMX-файлы, совместимые с этими инструментами.

### Количество батчей (Batch Count) {#batch-count}

Число батчей, обрабатываемых за одно действие «Align next». Может быть от 1 до 5. Обработка нескольких батчей за раз удобна для больших текстов, когда нет необходимости проверять каждый батч отдельно.

### Конфликт (Conflict) {#conflict}

Разрыв последовательности выравнивания, при котором алгоритм не смог установить чёткое отображение «один к одному» между исходными и целевыми предложениями. Конфликты возникают, когда переводчик разделяет, объединяет или переставляет предложения. Они обнаруживаются автоматически и отображаются в разделе «Conflicts» для разрешения.

### Корпус абзацев (Paragraph Corpora) {#paragraph-corpora}

Формат экспорта, в котором выровненные предложения группируются в абзацы на основе маркеров абзацев, обнаруженных при разбиении на предложения. Каждому абзацу исходного текста соответствует блок выровненных предложений целевого текста.

### Корпус предложений (Sentence Corpora) {#sentence-corpora}

Формат экспорта в виде текстовых файлов, содержащих по одному выровненному предложению на строку. Создаются два отдельных файла (по одному для каждого языка). Корпуса предложений широко используются в NLP-исследованиях, обучении машинного перевода и корпусной лингвистике.

### Косинусное сходство (Cosine Similarity) {#cosine-similarity}

Математическая мера угла между двумя векторами в диапазоне от -1 (противоположные) до 1 (идентичные). В Lingtrain косинусное сходство показывает, насколько семантически близки два эмбеддинга предложений. Более высокие значения указывают на большую семантическую близость. После L2-нормализации косинусное сходство равно скалярному произведению двух векторов.

### LaBSE {#labse}

Language-agnostic BERT Sentence Embeddings. Многоязычная модель эмбеддингов, разработанная Google, поддерживающая более 100 языков. LaBSE особенно эффективна для редких и малоресурсных языков, что делает её рекомендуемым выбором при выравнивании текстов на менее распространённых языковых парах.

### Лимит параллельных задач (Concurrent Task Limit) {#concurrent-task-limit}

Максимальное число задач выравнивания или разрешения конфликтов, которые пользователь может одновременно запустить. Лимит определяется тарифным планом пользователя и предотвращает перегрузку сервера. При достижении лимита новые задачи отклоняются до завершения текущих.

### Маска окна (Window Mask) {#window-mask}

Ограничение, применяемое к матрице сходства, обнуляющее значения сходства вне диагональной полосы, определённой параметром «окно». Это обеспечивает предположение о сохранении порядка предложений в переводе и предотвращает ложные совпадения из удалённых частей текста.

### Матрица сходства (Similarity Matrix) {#similarity-matrix}

Матрица, в которой элемент `[i][j]` содержит косинусное сходство между исходным предложением `i` и целевым предложением `j`. Вычисляется путём умножения матрицы исходных эмбеддингов на транспонированную матрицу целевых эмбеддингов. К ней применяется маска окна для ограничения совпадений диагональной полосой.

### Метрика качества (Quality Score) {#quality-score}

См. [Показатель цепочки](#chain-score).

### Модель эмбеддингов (Embedding Model) {#embedding-model}

Нейронная сеть, преобразующая текст в векторы эмбеддингов. Lingtrain поддерживает несколько многоязычных моделей, включая LaBSE, distiluse-base-multilingual-cased-v2, XLM-R-100langs, rubert-tiny2 и SONAR. Модели различаются охватом языков, скоростью и точностью.

### Обработка начала / конца (Handle Start / Handle Finish) {#handle-edges}

Опции при разрешении конфликтов, управляющие включением конфликтов в самом начале или конце выровненного текста в автоматическое разрешение. По умолчанию краевые конфликты исключаются, поскольку они часто вызваны различными предисловиями или послесловиями в исходном и целевом текстах.

### Окно (Window) {#window}

Число дополнительных предложений, добавляемых по обе стороны пропорционального диапазона целевого текста для батча. Большее окно увеличивает область поиска, повышая вероятность нахождения правильных соответствий при значительном различии числа предложений. Значение по умолчанию — 40-50.

### Очередь (Queue) {#queue}

Система обработки задач. При отправке задачи выравнивания или разрешения конфликтов она попадает в очередь, обрабатываемую воркерами Celery. Задачи обрабатываются в порядке приоритета. Позиция в очереди видна на странице выравнивания.

### Очистка (Erase) {#erase}

Операция, удаляющая все данные выравнивания (эмбеддинги, результаты батчей, историю правок) из выравнивания с сохранением записи о выравнивании и ссылок на исходные/целевые документы. После очистки можно выбрать другую модель эмбеддингов и выровнять текст заново.

### Очистка текста (Clean Text) {#clean-text}

Опция загрузки, применяющая базовую нормализацию текста: удаление лишних пробелов, исправление артефактов кодировки и стандартизацию переводов строк. Полезна для текстов, извлечённых из PDF или других источников с проблемами форматирования.

### Параллельный корпус (Parallel Corpus) {#parallel-corpus}

Набор данных, состоящий из выровненных пар предложений на двух языках. Параллельные корпуса — фундаментальные ресурсы для обучения машинного перевода, кросс-языковых NLP-исследований и переводоведения. Основная цель Lingtrain — создание параллельных корпусов из необработанных текстов.

### Пакетная обработка (Batch Processing) {#batch-processing}

Подход, при котором текст делится на сегменты фиксированного размера (батчи) и выравнивается последовательно. Это предотвращает исчерпание памяти на больших текстах, позволяет отслеживать прогресс и проверять качество каждого батча.

### Подстрочник (Proxy Text) {#proxy-text}

Машинный перевод разбитого текста, используемый как промежуточный текст при выравнивании. Когда модель эмбеддингов имеет ограниченные обучающие данные для языка, подстрочник на хорошо поддерживаемом языке (например, английском) может улучшить качество выравнивания. Оригинальные тексты сохраняются в итоговом результате.

### Показатель цепочки (Chain Score) {#chain-score}

Метрика качества, вычисляемая как `1 - (разрывы / общее_число_строк)`, где `разрывы` — число непоследовательных скачков в последовательности выравнивания, а `общее_число_строк` — общее число выровненных строк. Значение 1.0 означает идеальное последовательное выравнивание. Значения выше 0.95 считаются хорошим качеством.

### Размер батча (Batch Size) {#batch-size}

Число предложений исходного текста в каждом батче. По умолчанию — 200. Это значение фиксируется для всего выравнивания при его создании. Соответствующее окно целевого текста рассчитывается пропорционально отношению числа целевых предложений к исходным.

### Разбиение на предложения (Sentence Splitting) {#sentence-splitting}

Этап предобработки, при котором текст разделяется на отдельные предложения. Lingtrain использует языковые правила разбиения: библиотеку `razdel` для русского/украинского/белорусского, регулярные выражения для китайского и японского, стандартный токенизатор для остальных языков. В процессе разбиения извлекаются теги разметки.

### Разрешение конфликтов (Conflict Resolution) {#conflict-resolution}

Процесс исправления конфликтов выравнивания. Lingtrain предлагает автоматическое разрешение (перебор всех возможных группировок предложений и выбор лучшей по показателю сходства) и ручное разрешение в редакторе. Автоматический алгоритм обрабатывает малые конфликты (примерно до 6x6 предложений); более крупные требуют ручного вмешательства.

### Сдвиг (Shift) {#shift}

Параметр, вручную смещающий позицию окна целевого текста при обработке батчей. Положительные значения сдвигают окно вперёд (к более поздним предложениям); отрицательные — назад. Используйте сдвиг для компенсации, когда пропорция исходных и целевых предложений меняется по ходу текста, вызывая «дрейф» диагонали.

### SONAR {#sonar}

Массово многоязычный кодировщик предложений, разработанный Meta AI. SONAR поддерживает более 200 языков, обеспечивая самый широкий языковой охват среди доступных в Lingtrain моделей. Рекомендуется для чрезвычайно малоресурсных языков, недостаточно охваченных LaBSE.

### Тег разметки (Markup Tag) {#markup-tag}

Специальная аннотация в конце строки текста с префиксом `%%%%%` (например, `%%%%%author.`, `%%%%%h1.`). Теги разметки обозначают метаданные: имена авторов, заголовки, подзаголовки, цитаты и разделители. Помеченные строки исключаются из выравнивания и используются для форматирования при экспорте книг.

### TMX {#tmx}

Translation Memory eXchange. Стандартный XML-формат для хранения переводческой памяти. TMX-файлы содержат выровненные пары предложений «исходный — целевой» с метаданными языков и совместимы со всеми основными CAT-инструментами (SDL Trados, memoQ, OmegaT и др.).

### Цепочка (Chain) {#chain}

Последовательность подряд идущих выровненных пар предложений, в которой позиции целевых предложений монотонно возрастают. Цепочки представляют правильно выровненные области. Промежутки между цепочками — это конфликты. Более длинные цепочки свидетельствуют о более высоком качестве выравнивания.

### Экспорт (Export) {#export}

Процесс генерации скачиваемых файлов из выравнивания. Доступные форматы: параллельные HTML-книги, переводческая память TMX, корпуса предложений, корпуса абзацев, XML, JSON и нативный формат `.lt` базы выравнивания.

### Эмбеддинг (Embedding) {#embedding}

Плотный числовой вектор (обычно 768 измерений), представляющий семантический смысл предложения. Многоязычные модели эмбеддингов кодируют предложения из разных языков в одно и то же векторное пространство, что позволяет вычислять межъязыковое сходство с помощью косинусного сходства или скалярного произведения.

### HTML-книга (HTML Book) {#html-book}

Формат экспорта, создающий оформленную параллельную книгу для просмотра в любом веб-браузере. Книга отображает предложения исходного и целевого текстов рядом, с заголовками глав, указанием автора и стилями подсветки. Доступны три стиля: Simple, Pastel Fill и Pastel Start.

### Lingtrain Aligner (библиотека) {#lingtrain-aligner-library}

Python-библиотека (`lingtrain-aligner`), реализующая основной алгоритм выравнивания. Она обрабатывает разбиение на предложения, вычисление эмбеддингов, построение матрицы сходства, выбор лучших соответствий и разрешение конфликтов. Веб-приложение оборачивает эту библиотеку пользовательским интерфейсом, очередью задач и базой данных.

### Magic Alignment {#magic-alignment}

Экспериментальный режим выравнивания с усиленной плотностью поиска. При включении алгоритм применяет более плотный шаблон поиска по матрице сходства, что может давать лучшие результаты для текстов со значительными структурными различиями, но ценой увеличенного времени вычислений.

### Celery {#celery}

Распределённая система очередей задач, используемая Lingtrain для асинхронной обработки заданий выравнивания. Когда вы нажимаете «Align next» или «Resolve all», задача отправляется в очередь воркеров Celery. Это позволяет веб-интерфейсу оставаться отзывчивым, пока тяжёлые вычисления выполняются в фоновом режиме.
