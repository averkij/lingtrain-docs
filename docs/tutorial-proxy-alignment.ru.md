# Руководство: выравнивание с подстрочником {#proxy-alignment}

Не все языковые пары одинаково хорошо работают с моделями эмбеддингов по умолчанию. При выравнивании текстов на малоресурсных или далёких друг от друга языках прямое выравнивание может давать много конфликтов, разорванную диагональ на визуализации или просто некачественные сопоставления. Функция подстрочного перевода решает эту проблему с помощью машинного перевода в качестве посредника. Это руководство проведёт вас через реальный сценарий использования.

## Когда использовать подстрочный перевод {#when-to-use}

Используйте подстрочный перевод, когда:

- **Один или оба языка малоресурсные** — языки с ограниченным представительством в обучающих данных модели эмбеддингов. Примеры: башкирский, чувашский, татарский, якутский, бурятский, а также многие африканские и южноазиатские языки.
- **Прямое выравнивание даёт плохие результаты** — много конфликтов, разбросанные точки на визуализации вместо чистой диагонали, или явно неправильные сопоставления предложений.
- **Языковая пара далёкая** — модель может затрудняться с очень разными языковыми семьями (например, японский-финский, корейский-суахили).
- **Вы работаете с нестандартным письмом** — некоторые модели лучше обрабатывают определённые системы письма.

Подстрочник обычно **не нужен** для хорошо поддерживаемых пар: английский-русский, английский-немецкий, английский-французский, английский-испанский, английский-китайский и других основных комбинаций.

## Реальный сценарий: выравнивание чувашского и русского {#scenario}

Рассмотрим практический пример. Представьте, что вы работаете над проектом цифровой сохранности чувашского языка и у вас есть:

- Рассказ на **чувашском** языке (оригинал).
- Тот же рассказ в переводе на **русский**.

Вы хотите создать параллельную чувашско-русскую книгу. Однако, поскольку чувашский — малоресурсный язык, модель эмбеддингов по умолчанию имеет ограниченные обучающие данные для него, и прямое выравнивание может дать плохие результаты.

Решение: перевести разбитый чувашский текст на русский с помощью сервиса машинного перевода и использовать этот перевод как подстрочник при выравнивании.

## Шаг 1: Загрузка и разбиение текстов {#step-upload}

Начните с загрузки обоих текстов на вкладке **Documents** так же, как для обычного выравнивания:

1. Установите язык **From** — Chuvash, язык **To** — Russian.
2. Загрузите файл чувашского текста в левую панель.
3. Загрузите файл русского текста в правую панель.
4. Добавьте теги разметки (`%%%%%author.`, `%%%%%title.`) в оба файла перед загрузкой.

Проверьте предпросмотр предложений для обоих документов. Убедитесь, что разбиение выглядит правильно и теги разметки совпадают.

## Шаг 2: Создание выравнивания {#step-create}

Перейдите на вкладку **Alignments** и создайте новое выравнивание:

1. Нажмите **«+ Create alignment»**.
2. Дайте описательное имя (например, «Чувашский рассказ CV-RU»).
3. Выберите чувашский документ как исходный и русский как целевой.
4. Нажмите **«Create alignment»**.

Откройте страницу деталей выравнивания. Пока **не запускайте** выравнивание — сначала нужно подготовить подстрочник.

## Шаг 3: Скачивание разбитого текста {#step-download-split}

Перед созданием подстрочника нужна разбитая версия текста. В разбитой версии каждая строка — отдельное предложение, именно так его будет обрабатывать алгоритм выравнивания.

Вернитесь на вкладку **Documents**:

1. Нажмите на чувашский документ, чтобы открыть предпросмотр предложений.
2. Нажмите кнопку **download** в панели предпросмотра.
3. Сохраните скачанный файл — он содержит по одному предложению на строку с сохранёнными тегами разметки.

Это файл, который вы будете переводить для создания подстрочника.

## Шаг 4: Перевод разбитого текста {#step-translate}

Переведите скачанный разбитый текст с помощью любого сервиса машинного перевода. Ключевое требование: **перевод должен содержать ровно столько же строк**, сколько в оригинальном разбитом тексте. Каждая строка в подстрочнике должна соответствовать предложению на той же позиции в оригинале.

### Google Translate {#google-translate}

1. Откройте Google Translate в браузере.
2. Установите язык исходника — чувашский (если доступен) или автоопределение.
3. Установите язык перевода — русский.
4. Вставьте содержимое файла разбитого текста.
5. Скопируйте переведённый результат.
6. Сохраните как новый файл `.txt` (например, `chuvash_interlinear_ru.txt`).

### DeepL {#deepl}

1. Откройте DeepL Translator.
2. Загрузите файл разбитого текста или вставьте его содержимое.
3. Выберите чувашский (если поддерживается) как исходный и русский как целевой язык.
4. Скачайте или скопируйте переведённый текст.
5. Сохраните как файл `.txt`.

### Яндекс.Переводчик {#yandex}

Яндекс.Переводчик хорошо поддерживает тюркские языки, включая чувашский, башкирский и татарский, что делает его особенно хорошим выбором для этих языковых пар.

### Важно: проверка количества строк {#verify-lines}

После перевода убедитесь, что файл подстрочника содержит **ровно столько же строк**, сколько в оригинальном разбитом тексте. Откройте оба файла и сравните количество строк. Если переводчик объединил или разбил строки, нужно исправить подстрочник для восстановления построчного соответствия.

Частые проблемы:

- **Объединённые строки** — переводчик соединил две короткие строки в одну. Разделите их обратно.
- **Лишние пустые строки** — переводчик добавил пустые строки. Удалите их.
- **Пропущенные строки** — некоторые строки потерялись при переводе. Добавьте подстановочные переводы.

## Шаг 5: Загрузка подстрочника {#step-upload-proxy}

Вернитесь на страницу деталей выравнивания. Раздел **Подстрочник** показывает области загрузки для обоих языков — исходного («из») и целевого («в»).

### Загрузка подстрочного перевода {#upload-proxy-doc}

В нашем сценарии мы перевели **чувашский** (исходный) текст на русский. Поэтому загружаем подстрочник на стороне **«из»**:

1. Перетащите файл подстрочника (`chuvash_interlinear_ru.txt`) в область загрузки «Подстрочник (из)».
2. После загрузки статус меняется на **«Загружен»** с отображением имени файла.

Также можно:

- Нажать кнопку **download** для скачивания загруженного подстрочника.
- Нажать **open** для просмотра текста подстрочника в новой вкладке.
- Нажать **remove** для удаления и загрузки другой версии.

Если вы также хотите создать подстрочник для русского (целевого) текста — например, перевести его на английский — загрузите его на стороне «Подстрочник (в)». Это полезно при выравнивании двух малоресурсных языков через общий язык-посредник.

## Шаг 6: Включение подстрочника в настройках {#step-enable}

В панели **Settings** на странице деталей выравнивания:

1. Отметьте флажок **«Подстрочник (из)»** для включения чувашского подстрочника.
2. Если вы также загрузили целевой подстрочник, отметьте **«Подстрочник (в)»**.

Эти флажки становятся активными только после загрузки соответствующих подстрочных переводов.

## Шаг 7: Запуск выравнивания {#step-run}

Теперь запустите выравнивание как обычно:

1. Нажмите **«Align next»** для обработки первого пакета.
2. Дождитесь завершения обработки.
3. Проверьте визуализацию — вы должны увидеть гораздо более чистую диагональ по сравнению с тем, что дало бы прямое выравнивание.
4. Продолжите с оставшимися пакетами.

Когда подстрочник включён, система вычисляет эмбеддинги из подстрочных переводов вместо оригинальных текстов. Но оригинальные тексты сохраняются в результате выравнивания — подстрочник используется только на этапе сопоставления.

## Шаг 8: Проверка результатов {#step-verify}

После выравнивания:

1. **Проверьте визуализацию** — диагональ должна быть чистой. Если она всё ещё разорвана, качество подстрочного перевода может быть недостаточным, или тексты имеют структурные различия.
2. **Разрешите конфликты** — нажмите «Resolve all» как обычно.
3. **Просмотрите в редакторе** — включите **«Показать подстрочник»**, чтобы увидеть подстрочные аннотации под каждым предложением. Это помогает проверить качество выравнивания, даже если вы не читаете на чувашском.

Подстрочные аннотации появляются как мелкий текст под каждым предложением, показывая машинный перевод, использованный для выравнивания. Сравните их с русским целевым текстом, чтобы убедиться, что пары семантически корректны.

## Подстрочник для обеих сторон {#both-sides}

В некоторых случаях может понадобиться использовать подстрочник для обоих языков. Это полезно, когда:

- Оба языка малоресурсные (например, чувашский и башкирский).
- У вас есть общий язык-посредник (например, русский), на который можно перевести оба текста.

Процесс тот же — скачайте разбитые тексты для обоих документов, переведите оба на язык-посредник, загрузите оба подстрочника и включите оба флажка.

## Советы для лучших результатов {#tips}

1. **Выбирайте лучший доступный сервис перевода** для вашей языковой пары. Яндекс.Переводчик часто лучше работает для тюркских и славянских языков; Google Translate имеет более широкое покрытие; DeepL превосходен для европейских языков.
2. **Проверяйте количество строк** подстрочника. Несовпадение количества строк приведёт к ошибке загрузки или некорректному выравниванию.
3. **Используйте хорошо поддерживаемый язык-посредник.** Русский или английский обычно работают лучше всего, в зависимости от того, к какому языку исходник ближе культурно и лингвистически.
4. **Проверяйте качество подстрочника на выборке.** Перед обработкой всего текста вручную проверьте несколько переведённых предложений. Плохое качество подстрочника приведёт к плохому выравниванию.
5. **Рассмотрите подстрочник только для одной стороны.** Если один язык хорошо поддерживается, а другой нет, подстрочник нужен только для малоресурсной стороны.
6. **Сохраняйте файл подстрочника.** Если понадобится повторное выравнивание или корректировка, наличие готового подстрочника избавит от повторения этапа перевода.

## Ограничения {#limitations}

- **Качество подстрочника зависит от качества машинного перевода.** Если сервис МП плохо переводит ваш язык, подстрочник может не помочь.
- **Количество строк должно совпадать точно.** Любое расхождение вызовет проблемы.
- **Подстрочник добавляет шаг** в рабочий процесс, но обычно это оправдано для малоресурсных языков.
- **Конечный результат содержит оригинальные тексты**, а не подстрочные переводы. Подстрочник используется только на этапе сопоставления.
