# Руководство: экспорт результатов {#export-formats}

Когда выравнивание завершено (или частично завершено), вкладка **Create** позволяет просмотреть и экспортировать результаты в различных форматах. Это руководство объясняет каждый формат экспорта, его область применения и настройки вывода.

## Доступ к вкладке Create {#accessing}

Перейдите на вкладку **Create** в Aligner. Вы увидите интерфейс экспорта.

![Вкладка Create](assets/screenshots/aligner-create-empty.png)

Вкладка Create содержит три раздела:

1. **Выбор выравнивания** — какое выравнивание экспортировать.
2. **Настройки и предпросмотр** — параметры вывода и генерация живого предпросмотра.
3. **Скачивание** — экспорт в любом из доступных форматов.

Выберите выравнивание из выпадающего списка. Если в выравнивании обработан хотя бы один пакет, элементы управления экспортом становятся доступны. Если выравнивание ещё в процессе, отображается сообщение о том, что экспорт станет доступен после выравнивания хотя бы одного пакета.

## Настройки экспорта {#settings}

Перед экспортом настройте параметры, определяющие внешний вид и поведение вывода.

### Структура абзацев {#paragraph-structure}

Выберите, чьи разрывы абзацев использовать в экспортированном результате:

- **From** — абзацы из исходного (левого) текста.
- **To** — абзацы из целевого (правого) текста.

Это определяет, как предложения группируются в абзацы. В оригинальных текстах границы абзацев отмечены на последнем предложении каждого абзаца. Экспорт использует эти метки с выбранной стороны.

Обычно, если вы создаёте книгу для читателей исходного языка, используйте абзацную структуру «from». Если целевая аудитория читает на целевом языке, используйте «to».

### Порядок языков {#language-order}

Какой язык отображать первым (слева) в параллельном виде:

- **From** — исходный язык слева, целевой справа.
- **To** — целевой язык слева, исходный справа.

### Стиль подсветки {#highlight-style}

Визуальное оформление параллельной книги. Влияет только на экспорт HTML-книги:

- **Without highlight** — без фоновых цветов. Чистый, минималистичный вид, подходящий для печати или профессионального использования.
- **Pastel** — сплошные пастельные фоны для каждого языка. Облегчает различение языков при чтении.
- **Gradient** — градиентные фоны, затухающие от левого края. Более тонкое визуальное разделение.

### Плотность чтения {#reading-density}

Управляет интервалом между парами предложений в предпросмотре и HTML-книге:

- **Comfortable** — больше пустого пространства между парами. Комфортнее для длительного чтения.
- **Compact** — меньше пустого пространства. Больше контента на экране, меньше прокрутки.

### Количество абзацев (предпросмотр) {#paragraph-count}

Управляет количеством абзацев, показываемых в предпросмотре. Увеличьте или уменьшите, чтобы увидеть больше или меньше книги перед скачиванием.

## Генерация предпросмотра {#preview}

Нажмите **«Generate preview»**, чтобы увидеть живое отображение параллельной книги с текущими настройками. Предпросмотр показывает, как именно будет выглядеть финальная HTML-книга.

Предпросмотр включает:

- **Титульную страницу** — сгенерированную из тегов разметки `%%%%%author.` и `%%%%%title.`.
- **Заголовки глав** — из тегов `%%%%%h1.` по `%%%%%h5.`.
- **Пары предложений** — бок о бок с настроенной подсветкой и порядком языков.
- **Разделители секций** — из тегов `%%%%%divider.`.
- **Цитаты** — из тегов `%%%%%qtext.` и `%%%%%qname.`.

Индикатор статуса показывает, актуален ли предпросмотр или нуждается в обновлении после изменения настроек:

- **«Preview up to date»** — предпросмотр соответствует текущим настройкам.
- **«Controls changed»** — настройки были изменены с момента последней генерации. Нажмите «Apply changes» для обновления.

Корректируйте настройки и пересоздавайте, пока результат вас не устроит.

## Форматы экспорта {#formats}

### HTML-книга {#html-book}

**Формат:** Самодостаточный HTML-файл со встроенными CSS-стилями.

**Применение:**
- Чтение параллельных текстов в браузере на компьютере, планшете или телефоне.
- Распространение среди студентов или языковых партнёров.
- Печать на бумаге (используйте стиль «Without highlight» для лучших результатов печати).

**Содержимое:**
- Титульная страница с автором и названием.
- Заголовки глав и структурные элементы.
- Пары предложений бок о бок с выбранным стилем подсветки.
- Адаптивная вёрстка для экранов всех размеров.

**Как использовать:** Нажмите кнопку скачивания рядом с «HTML book». Откройте скачанный файл `.html` в любом браузере. Интернет-подключение не требуется — файл полностью автономен.

### TMX-корпус {#tmx}

**Формат:** Translation Memory eXchange (`.tmx`) — основанный на XML стандартный формат хранения памяти переводов.

**Применение:**
- Импорт в CAT-инструменты (memoQ, SDL Trados, OmegaT, Memsource, SmartCAT).
- Создание памяти переводов из существующих переводов.
- Обмен переводческими данными с другими переводчиками или агентствами.

**Содержимое:**
- Выровненные пары предложений с метаданными о языках.
- Каждая пара обёрнута в элемент `<tu>` (единица перевода) с дочерними элементами `<tuv>` (вариант единицы перевода).

**Как использовать:** Нажмите «Download TMX». Импортируйте скачанный файл в CAT-инструмент как память переводов. Инструмент будет использовать эти пары для нечёткого совпадения и повторного использования при переводе.

### Корпус предложений {#sentence-corpora}

**Формат:** Два текстовых файла (`.txt`), по одному для каждого языка. Каждая строка содержит одно предложение, выровненное по номеру строки — строка 1 в файле исходника соответствует строке 1 в файле перевода.

**Применение:**
- Исследования в области NLP — обучение моделей машинного перевода, оценка качества выравнивания.
- Создание параллельных корпусов для академических исследований.
- Анализ данных и статистика по длинам предложений, словарному запасу и т.д.
- Входные данные для других инструментов обработки текста, ожидающих параллельный текст.

**Содержимое:**
- Чистый текст без форматирования, разметки или метаданных.
- Одно предложение на строку, строго выровненное по позиции.

**Как использовать:** Нажмите кнопки скачивания для корпусов исходного и целевого языков отдельно. Каждый скачивается как файл `.txt`.

### Корпус абзацев {#paragraph-corpora}

**Формат:** Два текстовых файла (`.txt`), по одному для каждого языка. Предложения сгруппированы в абзацы на основе выбранной структуры абзацев.

**Применение:**
- Исследования, требующие выравнивания на уровне абзацев, а не предложений.
- Создание корпусов, где важен абзацный контекст.
- Сравнение абзацной структуры между переводами.

**Содержимое:**
- Текст, сгруппированный по абзацам, с пустыми строками между абзацами.
- Границы абзацев из стороны, выбранной в настройке «Paragraph structure».

**Как использовать:** Нажмите кнопки скачивания для корпусов абзацев исходного и целевого языков.

### Структурированные форматы (XML и JSON) {#structured}

**Формат:** XML и JSON представления полной структуры книги.

**Применение:**
- Пользовательские конвейеры обработки — когда нужен программный доступ к данным выравнивания.
- Конвертация в другие форматы (EPUB, PDF, пользовательский HTML) с помощью своих скриптов.
- Анализ данных, требующий структурированных метаданных (заголовки, структура страниц, идентификаторы выравнивания).
- Интеграция с другими программными системами.

**Содержимое:**
- Полная структура книги с метаданными (название, автор, заголовки, разделители).
- Выровненные пары предложений с оригинальными идентификаторами строк.
- Группировки абзацев и структурные элементы.

**Как использовать:** Нажмите кнопку скачивания XML или JSON. Обработайте файл на предпочитаемом языке программирования.

### База данных выравнивания (.lt) {#alignment-db}

**Формат:** Проприетарный формат Lingtrain на основе SQLite (`.lt`).

**Применение:**
- **Резервное копирование** — сохранение полного снимка выравнивания со всеми метаданными, эмбеддингами и историей редактирования.
- **Повторный импорт** — загрузка файла `.lt` обратно в Lingtrain для продолжения работы или обмена с коллегами.
- **Миграция** — перенос проектов выравнивания между экземплярами Lingtrain.
- **Архивирование** — долгосрочное хранение работы по выравниванию.

**Содержимое:**
- Полная база данных выравнивания со всеми пакетами, конфликтами и историей разрешения.
- Эмбеддинги предложений для обоих текстов.
- Все правки и модификации редактора.
- Данные прокси-документов (если есть).
- Параметры конфигурации (размер пакета, окно, сдвиг, модель).

**Как использовать:** Нажмите «Download .lt». Для повторного импорта перейдите на вкладку Alignments и используйте раздел «Upload alignment» в нижней части страницы. Перетащите файл `.lt` для импорта.

### Корпус сходства {#similarity-corpora}

**Формат:** Текстовый файл с оценками сходства.

**Применение:**
- Исследование метрик качества выравнивания.
- Фильтрация выровненных пар по уровню уверенности.
- Обучение и оценка моделей выравнивания.

**Содержимое:**
- Оценки сходства для выровненных пар предложений.

## Выбор нужного формата {#choosing}

| Ваша цель | Рекомендуемый формат |
|---|---|
| Читать параллельную книгу | HTML-книга |
| Загрузить в CAT-инструмент | TMX-корпус |
| Обучить модель МП | Корпус предложений |
| Академическое исследование | Корпус предложений или абзацев |
| Создать приложение | Структурированные форматы (JSON/XML) |
| Сделать резервную копию | База данных выравнивания (.lt) |
| Поделиться с коллегой | База данных выравнивания (.lt) |
| Напечатать двуязычную книгу | HTML-книга (стиль Without highlight) |
| Анализировать качество | Корпус сходства |

## Советы {#tips}

1. **Генерируйте предпросмотр перед скачиванием.** Предпросмотр даёт быструю визуальную проверку качества и форматирования.
2. **Попробуйте разные стили подсветки.** Каждый стиль лучше работает в разных контекстах: экран vs. печать, повседневное vs. академическое чтение.
3. **Используйте абзацную структуру языка, на котором читаете.** Разрывы абзацев из основного языка чтения будут ощущаться более естественно.
4. **Скачайте резервную копию .lt** перед внесением серьёзных правок. Если что-то пойдёт не так, вы сможете импортировать заново и начать сначала.
5. **Для исследований экспортируйте и корпус предложений, и корпус абзацев.** Они служат разным аналитическим целям и быстро генерируются.
6. **TMX-файлы могут быть большими.** Для длинных текстов TMX-файл может занимать несколько мегабайт. Это нормально, CAT-инструменты справляются с этим.
