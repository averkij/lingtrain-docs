# Справочник по разметке {#markup-reference}

Lingtrain Aligner использует специальную систему разметки для обозначения структурных элементов в текстовых файлах. Данная страница является полным справочником по всем поддерживаемым тегам разметки.

## Синтаксис {#syntax}

Все теги разметки следуют единому шаблону:

```
Содержимое текста%%%%%тег.
```

- Тег размещается **в конце строки**, непосредственно после текстового содержимого
- Префикс — ровно пять знаков процента: `%%%%%`
- После имени тега ставится точка (`.`)
- После точки не должно быть завершающих пробелов
- Каждая помеченная строка содержит только один тег

## Влияние тегов на выравнивание {#alignment-effect}

Помеченные строки **исключаются из алгоритма выравнивания**. Они не преобразуются в эмбеддинги и не сопоставляются с предложениями другого языка. Это означает:

- Теги не занимают ёмкость батча
- Теги не порождают конфликтов
- Теги сохраняются как метаданные в базе выравнивания

Оба текста (исходный и целевой) должны иметь **совпадающие теги**. Например, если в английском тексте есть теги `%%%%%author.` и `%%%%%title.`, в русском переводе они тоже должны быть. Несовпадение количества тегов может вызвать проблемы при экспорте.

## Влияние тегов на экспорт {#export-effect}

При экспорте теги используются для форматирования:

- **HTML-книга**: теги формируют титульные страницы, заголовки глав, блоки цитат, разделители секций и другие структурные элементы
- **TMX**: теги исключаются из пар предложений (они не являются единицами перевода)
- **Корпуса предложений/абзацев**: помеченные строки исключаются
- **XML/JSON**: теги включаются как элементы метаданных в структурированный вывод
- **База выравнивания (.lt)**: теги хранятся в таблице marks

## Справочник по тегам {#tag-reference}

### %%%%%author. {#tag-author}

**Назначение:** Обозначает имя автора.

**Синтаксис:**
```
Агата Кристи%%%%%author.
```

**Влияние на выравнивание:** Строка исключается из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: отображается на титульной странице в оформленном блоке автора
- Оба текста должны содержать этот тег; имя автора может различаться между языками

**Пример:**
```
English: Agatha Christie%%%%%author.
Russian: Агата Кристи%%%%%author.
```

---

### %%%%%title. {#tag-title}

**Назначение:** Обозначает название книги или произведения.

**Синтаксис:**
```
Синяя герань%%%%%title.
```

**Влияние на выравнивание:** Строка исключается из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: отображается крупно на титульной странице
- Один из важнейших тегов для экспорта книги

**Пример:**
```
English: The Blue Geranium%%%%%title.
Russian: Синяя герань%%%%%title.
```

---

### %%%%%h1. — %%%%%h5. {#tag-headings}

**Назначение:** Уровни заголовков от 1 до 5 — для названий глав, частей, разделов и подразделов.

**Синтаксис:**
```
Глава первая%%%%%h1.
Часть I%%%%%h2.
Раздел 1.1%%%%%h3.
Подраздел А%%%%%h4.
Мелкий заголовок%%%%%h5.
```

**Влияние на выравнивание:** Строки исключаются из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: отображаются как HTML-заголовки (`<h1>` — `<h5>`) с соответствующим оформлением
- Создают визуальные разрывы глав/разделов в параллельной книге

**Пример:**
```
English: Chapter One%%%%%h1.
Russian: Глава первая%%%%%h1.

English: Part I. The Beginning%%%%%h2.
Russian: Часть I. Начало%%%%%h2.
```

**Рекомендации:**
- Используйте `h1` для крупных глав
- Используйте `h2` для частей или подглав
- Используйте `h3`–`h5` умеренно для более мелких подразделений
- Сохраняйте единообразие уровней заголовков между исходным и целевым текстами

---

### %%%%%qtext. {#tag-qtext}

**Назначение:** Обозначает текст цитаты или эпиграфа.

**Синтаксис:**
```
Быть или не быть — вот в чём вопрос%%%%%qtext.
```

**Влияние на выравнивание:** Строка исключается из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: оформляется как стилизованный блок цитаты (обычно курсивом, с отступом)
- Часто используется в паре с `%%%%%qname.` для указания авторства

---

### %%%%%qname. {#tag-qname}

**Назначение:** Атрибуция цитаты (автор цитируемого высказывания).

**Синтаксис:**
```
Уильям Шекспир%%%%%qname.
```

**Влияние на выравнивание:** Строка исключается из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: отображается как текст атрибуции под цитатой, обычно с выравниванием по правому краю или мелким шрифтом

**Пример:**
```
English:
The only way to do great work is to love what you do%%%%%qtext.
Steve Jobs%%%%%qname.

Russian:
Единственный способ делать великие дела — любить то, что делаешь%%%%%qtext.
Стив Джобс%%%%%qname.
```

---

### %%%%%image. {#tag-image}

**Назначение:** Ссылка на файл изображения для включения в экспортируемую книгу.

**Синтаксис:**
```
illustration_01.jpg%%%%%image.
```

**Влияние на выравнивание:** Строка исключается из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: генерирует тег `<img>`, ссылающийся на указанный файл изображения

**Примечания:** Поддержка изображений предназначена прежде всего для экспорта HTML-книги. Сам файл изображения не загружается через разметку — он указывается по имени. Оба текста должны иметь совпадающие теги image в соответствующих позициях.

---

### %%%%%translator. {#tag-translator}

**Назначение:** Указание переводчика текста.

**Синтаксис:**
```
Перевод Иванова И. И.%%%%%translator.
```

**Влияние на выравнивание:** Строка исключается из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: отображается на титульной странице под именем автора

**Пример:**
```
English: Translated by John Doe%%%%%translator.
Russian: Перевод Иванова И. И.%%%%%translator.
```

---

### %%%%%divider. {#tag-divider}

**Назначение:** Создаёт разделитель секций (визуальный разделитель между разделами внутри главы).

**Синтаксис:**
```
* * *%%%%%divider.
```

**Влияние на выравнивание:** Строка исключается из выравнивания предложений.

**Влияние на экспорт:**
- HTML-книга: отображается как горизонтальная линия или декоративный разделитель

**Пример:**
```
...конец предыдущего раздела.

* * *%%%%%divider.

Начало следующего раздела...
```

**Рекомендации:**
- Текстовое содержимое строки-разделителя (например, «* * *») обычно одинаково в обоих языках
- Оба текста должны иметь разделители в соответствующих позициях

## Панель Marks {#marks-panel}

После загрузки документов панель **Marks** внизу вкладки «Documents» отображает все обнаруженные теги разметки из обоих текстов рядом. Используйте эту панель для проверки:

1. **Оба текста содержат одинаковое количество каждого типа тегов** — например, если в английском тексте 12 тегов `h1`, в русском тоже должно быть 12
2. **Содержимое тегов соответствует друг другу** — имя автора присутствует на обоих языках, названия глав соответствуют
3. **Порядок тегов последователен** — теги идут в одинаковой последовательности в обоих текстах

Несовпадение количества тегов — распространённая причина проблем при экспорте. Панель Marks позволяет легко обнаружить расхождения до начала выравнивания.

## Полный пример {#complete-example}

Вот пример корректно размеченного текстового файла:

```
Агата Кристи%%%%%author.
Синяя герань%%%%%title.
Перевод Ивановой М. А.%%%%%translator.


Смысл жизни не в том, чтобы найти себя, а в том, чтобы создать себя%%%%%qtext.
Джордж Бернард Шоу%%%%%qname.


Глава 1. Начало%%%%%h1.

— Когда я был здесь в прошлом году… — начал сэр Генри Клитеринг и умолк.

Хозяйка дома, миссис Бантри, посмотрела на него с любопытством.

* * *%%%%%divider.

Экс-комиссар Скотленд-Ярда гостил у старых друзей.

Глава 2. Расследование%%%%%h1.

Она открыла дверь и заглянула внутрь.
```

## Лучшие практики {#best-practices}

1. **Всегда добавляйте теги `author` и `title`** — они формируют титульную страницу при экспорте HTML
2. **Сохраняйте единообразие тегов между текстами** — одинаковое количество каждого типа в обоих текстах
3. **Добавляйте теги заголовков для глав** — `h1` для основных глав, `h2` для подразделов
4. **Ставьте разделители на стыках разделов** — они улучшают визуальную структуру экспортированных книг
5. **Проверяйте панель Marks** после загрузки — убедитесь в совпадении количества и порядка тегов до начала выравнивания
6. **Не вкладывайте теги** — каждая строка может содержать только один тег
7. **Не добавляйте лишние пробелы** после точки тега — это может помешать распознаванию
8. **Используйте теги цитат умеренно** — `qtext` и `qname` предназначены для эпиграфов и выделенных цитат, а не для каждой цитаты в тексте
