# Справочник форматов экспорта {#export-formats-reference}

На этой странице приведены подробные технические спецификации каждого формата экспорта, поддерживаемого Lingtrain Aligner. Общий обзор процесса экспорта см. на странице [Экспорт](export.ru.md).

## Настройки экспорта {#export-settings}

Перед экспортом настройте параметры вывода на вкладке **«Create»**:

- **Структура абзацев** (`par_direction`) — определяет, маркеры абзацев какого текста используются для группировки предложений. Значения: `from` (исходный текст) или `to` (целевой текст).
- **Порядок языков** (`left_lang`) — какой язык отображается первым (слева) в параллельном выводе. Значения: `from` или `to`.
- **Стиль подсветки** (`style`) — визуальное оформление HTML-книги. Значения: `none` (Simple), `pastel_fill` (Pastel Fill), `pastel_start` (Pastel Start).

## HTML-книга {#html-book}

**Расширение:** `.html`

**Описание:** Самодостаточный HTML-файл, отображающий параллельную двуязычную книгу в любом веб-браузере. Книга показывает выровненные пары предложений в структурированном макете с опциональным оформлением.

**Структура:**

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>Название книги — Имя автора</title>
  <style>
    /* Встроенные CSS-стили */
  </style>
</head>
<body>
  <!-- Титульная страница (из тегов author/title) -->
  <div class="title-page">
    <div class="author">Имя автора</div>
    <div class="title">Название книги</div>
    <div class="translator">Указание переводчика</div>
  </div>

  <!-- Эпиграф (из тегов qtext/qname) -->
  <div class="epigraph">
    <div class="quote-text">Текст цитаты</div>
    <div class="quote-name">Атрибуция</div>
  </div>

  <!-- Заголовок главы (из тегов h1-h5) -->
  <h1>Название главы (исходный) / Название главы (целевой)</h1>

  <!-- Выровненные абзацы -->
  <div class="paragraph">
    <span style="...">Предложение 1. Предложение 2.</span>
    <span style="...">Предложение 1. Предложение 2.</span>
  </div>

  <!-- Разделитель секций (из тега divider) -->
  <hr class="divider">
</body>
</html>
```

**Стили подсветки:**

| Стиль | Визуальный эффект |
|-------|-------------------|
| Simple (`none`) | Без фоновых цветов; чистый текст на белом фоне |
| Pastel Fill (`pastel_fill`) | Сплошные пастельные фоны — зелёный для исходного, розовый для целевого (со скруглением) |
| Pastel Start (`pastel_start`) | Градиентные фоны, переходящие от цвета к прозрачному |

**Настройка:** Сгенерированный HTML использует инлайн-стили. Вы можете изменить CSS после скачивания для настройки цветов, шрифтов, отступов и макета.

**Применение:** Чтение двуязычных книг в браузере, печать, распространение среди изучающих языки, публикация на сайтах.

## TMX (Translation Memory eXchange) {#tmx}

**Расширение:** `.tmx`

**Описание:** XML-стандарт для хранения данных переводческой памяти. TMX-файлы совместимы со всеми основными CAT-инструментами.

**Версия:** TMX 1.4

**Структура:**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE tmx SYSTEM "tmx14.dtd">
<tmx version="1.4">
  <header
    creationtool="lingtrain-aligner"
    creationtoolversion="1.0"
    datatype="plaintext"
    segtype="sentence"
    adminlang="en"
    srclang="en"
    o-tmf="lingtrain"
  />
  <body>
    <tu>
      <tuv xml:lang="en">
        <seg>Текст исходного предложения.</seg>
      </tuv>
      <tuv xml:lang="ru">
        <seg>Текст целевого предложения.</seg>
      </tuv>
    </tu>
    <!-- Дополнительные единицы перевода -->
  </body>
</tmx>
```

**Основные элементы:**

| Элемент | Описание |
|---------|----------|
| `<tmx>` | Корневой элемент с атрибутом версии |
| `<header>` | Метаданные об инструменте создания и конфигурации языков |
| `<tu>` | Translation Unit — одна выровненная пара предложений |
| `<tuv>` | Translation Unit Variant — текст на конкретном языке |
| `<seg>` | Segment — собственно текст предложения |

**Языковые коды:** Используются коды языков, настроенные в выравнивании (например, `en`, `ru`, `de`, `zh`).

**Совместимые CAT-инструменты:** SDL Trados Studio, memoQ, Memsource (Phrase), OmegaT, Wordfast, CafeTran, Smartcat, MateCat и другие.

**Примечания:**
- Строки с тегами разметки (author, title, заголовки и т. д.) исключаются из TMX
- Многопредложенческие пары объединяются в один сегмент
- TMX-файл кодируется в UTF-8

## Корпус предложений {#sentence-corpora}

**Расширение:** `.txt`

**Описание:** Текстовые файлы с одним выровненным предложением на строку. Генерируются два отдельных файла — для исходного и целевого языков. Строка `N` в исходном файле соответствует строке `N` в целевом.

**Формат:**

Исходный файл (`sentences_en.txt`):
```
When I was down here last year, said Sir Henry Clithering, and stopped.
His hostess, Mrs Bantry, looked at him curiously.
The Ex-Commissioner of Scotland Yard was staying with old friends.
```

Целевой файл (`sentences_ru.txt`):
```
Когда я был здесь в прошлом году… — начал сэр Генри Клитеринг и умолк.
Хозяйка дома, миссис Бантри, посмотрела на него с любопытством.
Экс-комиссар Скотленд-Ярда гостил у старых друзей.
```

**Кодировка:** UTF-8

**Применение:** NLP-исследования, обучающие данные для машинного перевода, корпусная лингвистика, статистический анализ.

## Корпус абзацев {#paragraph-corpora}

**Расширение:** `.txt`

**Описание:** Текстовые файлы, сгруппированные по абзацам. Предложения внутри абзаца объединяются пробелами. Абзацы разделяются пустыми строками. Структура абзацев определяется настройкой `par_direction`.

**Применение:** Сравнение переводческих решений на уровне абзацев, анализ текстовой структуры.

## XML {#xml}

**Расширение:** `.xml`

**Описание:** Структурированное XML-представление выровненной книги, включающее метаданные, структуру и все выровненные пары предложений.

**Структура:**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<book>
  <head>
    <author from="Author Name" to="Имя Автора"/>
    <title from="Book Title" to="Название Книги"/>
  </head>
  <contents>
    <item>
      <h1 from="Chapter One" to="Глава первая"/>
    </item>
    <item>
      <from>Source sentence text.</from>
      <to>Target sentence text.</to>
    </item>
    <item>
      <divider/>
    </item>
  </contents>
</book>
```

**Основные элементы:**

| Элемент | Описание |
|---------|----------|
| `<book>` | Корневой элемент |
| `<head>` | Метаданные (автор, название, переводчик) |
| `<contents>` | Список элементов содержимого |
| `<item>` | Один элемент (пара предложений, заголовок или разделитель) |
| `<from>`, `<to>` | Исходный и целевой текст пары предложений |
| `<h1>`–`<h5>` | Заголовки с атрибутами `from` и `to` |

**Применение:** Пользовательские конвейеры обработки, трансформация данных, программный анализ.

## JSON {#json}

**Расширение:** `.json`

**Описание:** Структурированное JSON-представление выровненной книги, содержащее те же данные, что и XML, в JSON-нативной структуре.

**Структура:**

```json
{
  "head": {
    "author": {
      "from": "Author Name",
      "to": "Имя Автора"
    },
    "title": {
      "from": "Book Title",
      "to": "Название Книги"
    }
  },
  "contents": [
    {
      "type": "h1",
      "from": "Chapter One",
      "to": "Глава первая"
    },
    {
      "type": "text",
      "from": "Source sentence text.",
      "to": "Target sentence text."
    },
    {
      "type": "divider"
    }
  ]
}
```

**Кодировка:** UTF-8, с `ensure_ascii=False` для корректного представления Unicode.

**Применение:** Веб-приложения, обработка JavaScript, API-интеграции, анализ данных на Python/R.

## База выравнивания (.lt) {#lt-format}

**Расширение:** `.lt`

**Описание:** Нативный формат выравнивания Lingtrain. Это SQLite-база данных, содержащая полное состояние выравнивания. Может быть повторно импортирована в Lingtrain для восстановления выравнивания.

**Таблицы базы данных:**

| Таблица | Описание |
|---------|----------|
| `splitted_from` | Предложения исходного текста (одна строка на предложение) |
| `splitted_to` | Предложения целевого текста |
| `doc_index` | Выровненный индекс документа — соответствие между ID предложений |
| `batches` | Метаданные батчей (batch_id, shift, window) |
| `marks` | Теги разметки (тип, текст, индекс строки, направление языка) |
| `proxy_from` | Предложения подстрочника для исходного языка (если загружен) |
| `proxy_to` | Предложения подстрочника для целевого языка |
| `embeddings_from` | Кешированные эмбеддинги исходного текста (если включено) |
| `embeddings_to` | Кешированные эмбеддинги целевого текста |
| `metadata` | Метаданные «ключ-значение» (коды языков, информация о модели, размер батча) |

**Основные столбцы `doc_index`:**

| Столбец | Тип | Описание |
|---------|-----|----------|
| `id` | INTEGER | Индекс строки в выравнивании |
| `text_from` | TEXT | Объединённый текст исходных предложений |
| `from_ids` | TEXT | JSON-массив ID исходных строк (напр., `[0, 1]`) |
| `text_to` | TEXT | Объединённый текст целевых предложений |
| `to_ids` | TEXT | JSON-массив ID целевых строк |
| `batch_id` | INTEGER | К какому батчу относится эта пара |

**Примечания:**
- При скачивании BLOB-данные эмбеддингов могут быть удалены для уменьшения размера файла
- Файл `.lt` — стандартная база данных SQLite3, которую можно открыть любым клиентом SQLite (DB Browser for SQLite, CLI `sqlite3`, модуль Python `sqlite3`)
- При повторном импорте файла `.lt` создаётся новое выравнивание с теми же данными, но новым GUID

**Применение:** Резервное копирование, миграция между экземплярами Lingtrain, офлайн-анализ, пользовательские инструменты.
