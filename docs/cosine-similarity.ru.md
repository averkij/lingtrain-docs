# Понимание косинусного сходства {#cosine-similarity}

Косинусное сходство — математическая мера, лежащая в основе сопоставления предложений в Lingtrain Aligner. Оно количественно оценивает, насколько похожи два предложения, сравнивая направления их векторов-эмбеддингов. На этой странице мы объясняем концепцию с нуля, делая её доступной для читателей без математической подготовки, одновременно приводя точные формулы для тех, кому они нужны.

## Векторы: базовая идея {#vectors-basic}

### Что такое вектор? {#what-is-vector}

**Вектор** — это просто список чисел, представляющий точку или направление в пространстве. В повседневной жизни:

- Местоположение на карте описывается двумя числами: широтой и долготой. Это двумерный вектор.
- Цвет на экране описывается тремя числами: красный, зелёный, синий (RGB). Это трёхмерный вектор.
- Эмбеддинг предложения описывается сотнями чисел (768, 1024 или 1536 в зависимости от модели). Это многомерный вектор.

Каждое число в векторе называется **компонентой** или **измерением** (размерностью).

### Векторы в 2D {#vectors-2d}

Начнём с простых двумерных векторов для формирования интуиции. Рассмотрим два вектора:

- **A** = (3, 4) — направлен на 3 единицы вправо и 4 вверх
- **B** = (6, 8) — направлен на 6 единиц вправо и 8 вверх

Эти два вектора указывают в **одном направлении** — B есть A, умноженный на 2. Несмотря на разные модули (длины), их направление совпадает.

Теперь рассмотрим:

- **C** = (4, -3) — направлен на 4 единицы вправо и 3 вниз

Вектор C указывает в совершенно другом направлении, чем A. Он почти перпендикулярен.

Ключевое наблюдение: **направление отражает смысл, а модуль — нет.** Два предложения, выражающие одну и ту же мысль, должны указывать в одном направлении в пространстве эмбеддингов, даже если одно из них длиннее или детальнее другого.

### Векторы в 3D {#vectors-3d}

В 3D векторы имеют три компоненты: (x, y, z). Их можно представить как стрелки в трёхмерном пространстве. Те же принципы работают — направление важнее длины для измерения сходства.

```
        z
        |    B
        |   /
        |  /
        | /
        |/_______ y
       /
      /  A
     /
    x

    Векторы A и B указывают в похожих направлениях
    = высокое косинусное сходство
```

### Многомерные векторы {#high-dimensional}

Эмбеддинги предложений существуют в пространствах с сотнями измерений. Хотя визуализировать 768 измерений мы не можем, математика остаётся той же. Два вектора в 768-мерном пространстве могут быть:

- **Параллельны** (одно направление) — косинусное сходство близко к 1
- **Перпендикулярны** (не связаны) — косинусное сходство близко к 0
- **Противоположны** (противоположный смысл) — косинусное сходство близко к -1

На практике эмбеддинги предложений почти всегда имеют положительное косинусное сходство (от 0 до 1), поскольку модели обучены создавать векторы в положительном конусе пространства.

## Формула косинуса {#formula}

### Угол между векторами {#angle}

**Косинусное сходство** между двумя векторами A и B — это косинус угла между ними:

```
cosine_similarity(A, B) = cos(theta)
```

Где theta — угол между векторами A и B.

- Когда theta = 0 градусов (одно направление): cos(0) = 1
- Когда theta = 90 градусов (перпендикулярны): cos(90) = 0
- Когда theta = 180 градусов (противоположны): cos(180) = -1

### Вычисление через скалярное произведение {#dot-product}

Для вычисления косинуса без предварительного нахождения угла используется формула **скалярного произведения**:

```
cosine_similarity(A, B) = (A . B) / (||A|| * ||B||)
```

Где:

**Скалярное произведение** (A . B): перемножаем соответствующие компоненты и суммируем:
```
A . B = a1*b1 + a2*b2 + ... + an*bn
```

**Модуль** ||A||: длина вектора A:
```
||A|| = sqrt(a1^2 + a2^2 + ... + an^2)
```

### Разобранный пример в 2D {#example-2d}

Вычислим косинусное сходство между A = (3, 4) и B = (6, 8):

**Скалярное произведение:**
```
A . B = 3*6 + 4*8 = 18 + 32 = 50
```

**Модули:**
```
||A|| = sqrt(3^2 + 4^2) = sqrt(9 + 16) = sqrt(25) = 5
||B|| = sqrt(6^2 + 8^2) = sqrt(36 + 64) = sqrt(100) = 10
```

**Косинусное сходство:**
```
cos_sim = 50 / (5 * 10) = 50 / 50 = 1.0
```

Значение 1.0 означает, что векторы указывают в точно одном направлении — что логично, ведь B = 2*A.

Теперь попробуем A = (3, 4) и C = (4, -3):

**Скалярное произведение:**
```
A . C = 3*4 + 4*(-3) = 12 - 12 = 0
```

**Косинусное сходство:**
```
cos_sim = 0 / (5 * 5) = 0.0
```

Значение 0 означает, что векторы перпендикулярны — совершенно не связанные направления.

## Почему косинусное сходство для семантического сравнения {#why-cosine}

### Направление против модуля {#direction-vs-magnitude}

Рассмотрим два предложения:

- «Кот сидел на коврике.»
- «Кот сидел на коврике в углу комнаты рядом со старым камином.»

Второе предложение содержит больше информации, но основной смысл схож. В пространстве эмбеддингов вектор второго предложения может иметь больший модуль (больше содержания = больше «энергии» в векторе), но должен указывать приблизительно в том же направлении.

Косинусное сходство игнорирует различия в модулях и фокусируется на направлении, что делает его устойчивым к вариациям длины предложений. Это именно то, что нужно для выравнивания — короткое предложение должно соответствовать своему переводу вне зависимости от того, длиннее перевод или короче.

### Почему не евклидово расстояние? {#why-not-euclidean}

**Евклидово расстояние** измеряет расстояние по прямой между двумя точками:

```
distance = sqrt((a1-b1)^2 + (a2-b2)^2 + ... + (an-bn)^2)
```

Проблема: два вектора, указывающие в одном направлении, но с разными модулями, будут иметь большое евклидово расстояние, хотя они представляют схожие значения. Косинусное сходство корректно определяет их как похожие.

Однако когда векторы **L2-нормализованы** (масштабированы до единичной длины), евклидово расстояние и косинусное сходство становятся эквивалентными — оба ранжируют пары одинаково. Вот почему Lingtrain нормализует эмбеддинги: это позволяет использовать эффективное вычисление скалярного произведения, сохраняя семантическое ранжирование.

## L2-нормализация {#l2-normalization}

### Что она делает {#what-l2-does}

**L2-нормализация** масштабирует вектор так, чтобы его модуль стал ровно 1:

```
A_normalized = A / ||A||
```

Для A = (3, 4) с ||A|| = 5:
```
A_normalized = (3/5, 4/5) = (0.6, 0.8)
```

Проверка: ||A_normalized|| = sqrt(0.6^2 + 0.8^2) = sqrt(0.36 + 0.64) = sqrt(1.0) = 1.0

### Оптимизация через скалярное произведение {#dot-product-shortcut}

Когда оба вектора нормализованы, формула косинусного сходства резко упрощается:

```
cosine_similarity(A_norm, B_norm) = A_norm . B_norm
```

Знаменатель (||A|| * ||B||) становится 1 * 1 = 1, поэтому он исчезает. Это значит, что вычисление косинусного сходства между нормализованными векторами — просто скалярное произведение, самая простая и быстрая векторная операция.

### Почему это важно для производительности {#performance}

В Lingtrain алгоритм выравнивания вычисляет сходство между каждым исходным и каждым целевым предложением в пакете. Для пакета из 200 исходных и 250 целевых предложений это 50 000 вычислений сходства.

С нормализованными векторами все 50 000 вычислений происходят в одном матричном умножении:

```
матрица_сходства = исходные_эмбеддинги @ целевые_эмбеддинги.T
```

Эта операция высоко оптимизирована в числовых библиотеках (NumPy, BLAS) и завершается за миллисекунды. Без нормализации каждое вычисление потребовало бы дополнительных расчётов модулей, значительно замедляя процесс.

## Практическая интерпретация оценок сходства {#interpretation}

В контексте выравнивания предложений оценки косинусного сходства обычно попадают в следующие диапазоны:

| Диапазон | Интерпретация |
|----------|--------------|
| 0.85 - 1.0 | Очень высокое сходство — почти наверняка переводная пара |
| 0.70 - 0.85 | Высокое сходство — вероятно, перевод, возможно с некоторой перестройкой |
| 0.50 - 0.70 | Умеренное сходство — может быть связано, но не прямой перевод |
| 0.30 - 0.50 | Низкое сходство — вероятно, разные предложения на близкую тему |
| 0.0 - 0.30 | Очень низкое сходство — не связанные предложения |

Эти диапазоны приблизительны и зависят от модели эмбеддингов, языковой пары и типа текста. Алгоритм выравнивания не использует фиксированных порогов — он выбирает лучшее соответствие для каждого исходного предложения, какой бы ни была оценка.

## Связь с алгоритмом выравнивания {#connection-to-alignment}

В конвейере выравнивания Lingtrain косинусное сходство используется в трёх местах:

1. **Матрица сходства** — основной шаг, на котором каждое исходное предложение сравнивается с каждым кандидатным целевым. Пара с наивысшей оценкой становится соответствием в выравнивании.

2. **Разрешение конфликтов** — при разрешении неоднозначностей группировки (например, должны ли исходные предложения 5-6 соответствовать целевым 7-8-9 как группа 2:3?) объединённые эмбеддинги сравниваются с помощью косинусного сходства для оценки каждой возможной группировки.

3. **Оконная маска** — матрица сходства маскируется так, чтобы учитывались только оценки в пределах диагональной полосы. Это предотвращает некорректное сопоставление далёких предложений, даже если они случайно имеют высокое косинусное сходство.

Полные детали алгоритма см. в [Алгоритм выравнивания](algorithm.ru.md).
