# Руководство автора курсов {#creator-guide}

Это руководство описывает каждый этап создания, редактирования и публикации курса в Lingtrain. Студия курсов предоставляет рабочее пространство с тремя панелями для построения структурированного учебного контента с модулями и юнитами.

## Обзор {#overview}

**Студия курсов** -- среда разработки для авторов курсов. Она доступна по адресу `/workspace/courses` и предоставляет инструменты для:

- Создания курсов с двуязычными названиями, описаниями и метаданными
- Организации контента в модули и юниты
- Редактирования содержимого юнитов через визуальный редактор форм или JSON-редактор
- Импорта контента из JSON-файлов (отдельные юниты или полные пакеты курсов)
- Генерации контента с помощью ИИ
- Валидации и публикации курсов для учеников

Студия использует трёхпанельный интерфейс: **структура курса** слева, **редактор** в центре и **панель валидации** справа. Обе боковые панели сворачиваются.

## Создание курса {#creating-a-course}

1. Перейдите в **Рабочее пространство > Курсы** (`/workspace/courses`).
2. Нажмите **Создать курс**.
3. Заполните настройки курса:
   - **Название** (EN и/или RU) -- хотя бы один язык обязателен
   - **Описание** (EN и/или RU) -- краткое описание, отображаемое в каталоге
   - **Область** -- выберите: `language-learning`, `programming`, `general`, `writing`
   - **Сложность** -- `beginner`, `intermediate` или `advanced`
   - **Обложка** -- необязательное изображение для карточки курса
   - **Теги** -- необязательные ключевые слова для фильтрации в каталоге
4. Нажмите **Сохранить**. Новый курс откроется в студии.

## Добавление модулей и юнитов {#adding-modules-units}

Курсы организованы в трёхуровневую иерархию: **Курс > Модули > Юниты**.

### Модули {#modules}

- Нажмите **Добавить модуль** в структуре курса (левая панель).
- Укажите название модуля. Модули выполняют роль глав или разделов.
- Перетаскивайте модули для изменения порядка в программе курса.

### Юниты {#units}

- Выберите модуль, затем нажмите **Добавить юнит**.
- Выберите **тип** юнита:

| Тип | Назначение |
|-----|-----------|
| `lesson` | Учебный контент с текстом, медиа и упражнениями |
| `reference` | Справочный материал (глоссарий, таблицы) |
| `parallel_reading` | Параллельный двуязычный текст из снимка выравнивания |
| `drill` | Практические упражнения с фокусом на активности |
| `assessment` | Оцениваемый тест с подсчётом баллов и лимитом попыток |
| `project` | Открытое задание с инструкциями |

- Перетаскивайте юниты для изменения порядка внутри модуля или между модулями.
- Нажмите на юнит, чтобы открыть его в редакторе.

## Визуальный редактор форм {#form-editor}

Вкладка **Форма** предоставляет визуальный редактор для создания контента юнитов без написания JSON.

### Блоки контента {#content-blocks}

Нажмите **Добавить блок** для вставки контента. Доступные типы блоков:

| Тип блока | Описание |
|-----------|----------|
| `rich_text` | Форматированный текст с поддержкой Markdown |
| `callout` | Выделенный блок с тоном: `info`, `tip`, `warning`, `danger` |
| `term_grid` | Фильтруемая сетка терминов с определениями |
| `dialogue` | Диалог в формате чат-пузырей между участниками |
| `code` | Блок кода с подсветкой синтаксиса и выбором языка |
| `parallel_reader` | Двуязычный текст из снимка выравнивания |

Каждый блок имеет уникальный `block_id`, генерируемый автоматически. Используйте ручку перетаскивания для изменения порядка блоков.

### Активности {#activities}

Нажмите **Добавить активность** для вставки интерактивных упражнений. Доступные типы:

| Тип активности | Описание |
|----------------|----------|
| `multiple_choice` | Вопрос с одним или несколькими вариантами ответа |
| `match_pairs` | Упражнение на сопоставление пар перетаскиванием или нажатием |
| `fill_in_blank` | Текст с пропусками для заполнения учеником |
| `order_sequence` | Расстановка элементов в правильном порядке перетаскиванием |
| `flashcards` | Двусторонние карточки с 3D-анимацией переворота |
| `short_text` | Свободный ввод текста с проверкой по допустимым ответам |

Активности оцениваются автоматически серверным движком. Юниты типов drill и assessment сфокусированы на активностях, а lesson обычно сочетает блоки контента с активностями.

### Правила завершения {#form-completion-rules}

Настройте, как юнит определяет прохождение учеником:

- **Все обязательные** -- ученик должен выполнить каждую активность
- **Порог баллов** -- ученик должен набрать минимальный процент баллов
- **Смешанный** -- объединяет оба требования

## JSON-редактор {#json-editor}

Вкладка **JSON** показывает необработанный payload формата `course-unit.v1`. Это полезно для:

- Вставки контента из внешних инструментов
- Точной настройки полей, не доступных в визуальном редакторе
- Отладки ошибок валидации

Редактор валидирует JSON в реальном времени. Ошибки отображаются прямо в редакторе и также перечислены в панели валидации (правая панель).

Полную документацию по схеме см. в [Справочнике JSON-формата курсов](course-json-reference.ru.md).

## Импорт контента {#importing-content}

### Импорт одного юнита {#import-unit}

1. Откройте редактор юнита и нажмите **Импорт** на панели инструментов.
2. Выберите файл `.json`, содержащий payload формата `course-unit.v1`.
3. Система валидирует файл (лимит 256 КБ) и показывает предварительный просмотр ошибок и предупреждений.
4. Подтвердите для замены текущего содержимого юнита.

### Импорт пакета курса {#import-bundle}

1. На странице списка курсов нажмите **Импортировать курс**.
2. Выберите файл `.json`, содержащий payload формата `course-bundle.v1`.
3. Система создаст новый черновик курса со всеми модулями и юнитами из пакета.
4. Проверьте импортированный контент в студии перед публикацией.

Формат пакета описан в [справочнике JSON](course-json-reference.ru.md#bundle-schema).

## Генерация контента с ИИ {#ai-generation}

Студия интегрирует ИИ для помощи в генерации учебного контента. Все функции ИИ требуют активной подписки с доступными кредитами на ИИ-операции.

### Генерация юнита {#ai-generate-unit}

1. Откройте юнит в редакторе и нажмите **Сгенерировать с ИИ** на панели инструментов.
2. Заполните структурированную форму:
   - **Тема** -- что должен объяснять юнит
   - **Область** -- целевая предметная область
   - **Тип** -- тип юнита (lesson, drill, assessment и др.)
   - **Сложность** -- beginner, intermediate или advanced
   - **Исходный текст** (необязательно) -- вставьте существующий контент как основу
   - **Модификатор** (необязательно) -- свободная текстовая инструкция для ИИ
3. Нажмите **Сгенерировать**. ИИ создаст payload формата `course-unit.v1`.
4. Просмотрите результат в **двойном предварительном просмотре**:
   - Вкладка **Просмотр** -- показывает, как ученики увидят контент
   - Вкладка **JSON** -- показывает необработанный payload со статусом валидации
5. Выберите действие:
   - **Использовать** -- импортирует контент в редактор как черновик
   - **Редактировать** -- переключает на вкладку JSON для ручных правок
   - **Перегенерировать** -- генерирует новую версию

### Генерация пакета курса {#ai-generate-bundle}

1. На странице списка курсов (`/workspace/courses`) нажмите **Сгенерировать курс с ИИ**.
2. Заполните ту же структурированную форму плюс количество модулей (1--3).
3. ИИ сгенерирует полный `course-bundle.v1` с модулями и юнитами.
4. При успехе создаётся новый черновик курса. Нажмите **Открыть в студии** для проверки.

### Исправить с ИИ {#ai-fix}

Когда JSON-редактор показывает ошибки валидации:

1. Нажмите **Исправить с ИИ** (появляется на панели инструментов JSON-вкладки при наличии ошибок).
2. Система отправляет некорректный JSON в сервис нормализации ИИ.
3. Просмотрите исправленный JSON в диалоге подтверждения.
4. Нажмите **Применить** для замены содержимого редактора.

### Требование проверки ИИ-контента {#ai-review}

Весь контент, сгенерированный ИИ, сохраняется как **черновик** с флагом `ai_generated`, установленным в `true`. Этот флаг **блокирует публикацию** до тех пор, пока вы не проверите и не отредактируете контент:

- Панель валидации показывает предупреждение: *"N юнитов, сгенерированных ИИ, требуют проверки"*
- Любое сохранение в режиме Формы или JSON **автоматически сбрасывает** флаг
- Это гарантирует, что каждый юнит, сгенерированный ИИ, был проверен человеком перед публикацией

## Публикация {#publishing}

### Валидация {#validation}

Перед публикацией панель валидации (правая панель) проверяет:

- Отсутствие обязательных полей (название, версия схемы)
- Некорректные типы блоков или активностей
- Превышение лимита размера (256 КБ на юнит)
- Юниты, сгенерированные ИИ, требующие проверки
- Отсутствие учебных целей и описаний (предупреждения)

Исправьте все **ошибки** перед публикацией. Предупреждения носят информационный характер.

### Создание релиза {#creating-release}

1. Убедитесь, что все ошибки валидации исправлены.
2. Нажмите **Опубликовать** в настройках курса.
3. Система создаст неизменяемый **снимок релиза**, который зафиксирует весь контент, включая ссылки на параллельные тексты.
4. Опубликованный курс появится в каталоге для учеников.

Ученики всегда видят опубликованный релиз, а не живой черновик. Вы можете продолжать редактирование после публикации -- изменения появятся только при создании нового релиза.

## Советы {#tips}

- **Шаблоны** -- используйте диалог шаблонов при создании нового юнита, чтобы начать с готовой структуры вместо пустого холста.
- **Режим предпросмотра** -- вкладка Просмотр показывает точно, как ученики увидят юнит, включая интерактивные активности.
- **Автосохранение** -- студия автоматически сохраняет вашу работу по мере редактирования. Индикатор автосохранения на панели инструментов показывает текущий статус сохранения.
- **Навигация по клавиатуре** -- используйте структуру курса для быстрого переключения между юнитами без выхода из студии.
- **Перенос между модулями** -- перетащите юнит из одного модуля в другой для реорганизации программы курса.
