# Параметры окна и смещения {#window-and-shift}

Параметры **окна (window)** и **смещения (shift)** — это два наиболее важных параметра выравнивания в Lingtrain Aligner. Они управляют тем, где алгоритм ищет совпадения в целевом тексте, напрямую влияя на точность выравнивания. Понимание этих параметров необходимо для получения хороших результатов, особенно на текстах, отклоняющихся от простого попредложенческого соответствия один к одному.

## Математическое определение {#math}

Для каждого предложения источника с индексом `i` в пакете инструмент выравнивания ищет наилучшее совпадение среди предложений цели в диапазоне, определяемом окном и смещением:

```
target_range = [i + shift - window, i + shift + window]
```

Где:
- `i` — индекс предложения источника (внутри текущего пакета)
- `shift` определяет центр диапазона поиска относительно `i`
- `window` определяет полуширину диапазона поиска

Например, при `window = 4` и `shift = 0` предложение 10 источника будет сравниваться с предложениями цели с 6-го по 14-е (10 - 4 = 6, 10 + 4 = 14). Инструмент вычисляет косинусное сходство между эмбеддингом предложения источника и эмбеддингом каждого предложения цели в этом диапазоне, а затем выбирает лучшее совпадение.

## Параметр окна {#window}

### Что он делает {#window-what}

Окно определяет, насколько далеко от ожидаемой позиции инструмент будет искать совпадения. Окно размером `n` означает, что инструмент рассматривает `2n + 1` предложений цели для каждого предложения источника (n до и n после ожидаемой позиции, плюс сама ожидаемая позиция).

### Значение по умолчанию {#window-default}

Размер окна по умолчанию обычно составляет 4–6, что хорошо подходит для большинства хорошо структурированных параллельных текстов. Это означает, что инструмент просматривает примерно 9–13 предложений цели для каждого предложения источника.

### Когда увеличивать окно {#window-increase}

Увеличивайте окно, когда:

- **Перевод имеет структурные различия**: Переводчик значительно перестроил текст, разбивая, объединяя и переставляя предложения. Большее окно даёт инструменту больше пространства для нахождения верного совпадения, даже если предложения смещены.
- **Число предложений сильно различается**: Если источник содержит 100 предложений, а цель — 130, смещение накапливается, и предложения ближе к концу пакета могут оказаться за пределами окна по умолчанию.
- **Свободный или литературный перевод**: Литературные переводчики часто позволяют себе вольности с порядком предложений. Большее окно компенсирует это.
- **Визуализация показывает дрейф**: Если диаграмма рассеяния показывает, что точки постепенно удаляются от диагонали, окно может быть слишком маленьким для захвата верных совпадений.

### Когда уменьшать окно {#window-decrease}

Уменьшайте окно, когда:

- **Тексты жёстко структурированы**: Техническая документация, юридические тексты и субтитры обычно имеют очень тесное структурное соответствие. Меньшее окно снижает шанс ложноположительных совпадений.
- **Важна скорость**: Меньшее окно означает меньше вычислений сходства на предложение, что ускоряет выравнивание.
- **Появляются ложные совпадения**: Если инструмент сопоставляет явно неверные предложения (например, предложение о теме А с предложением о теме Б, случайно оказавшимся рядом), уменьшение окна может помочь, ограничив пространство поиска.

### Компромиссы {#window-tradeoffs}

| Размер окна | Преимущества | Недостатки |
|-------------|-------------|------------|
| Малое (2–3) | Быстро; меньше ложных совпадений | Может пропустить верные совпадения при различиях в структуре |
| Среднее (4–6) | Хороший баланс для большинства текстов | По умолчанию; работает для типичных переводов |
| Большое (8–15) | Хорошо справляется со структурными различиями | Медленнее; больше риск ложных совпадений; выше потребление памяти |
| Очень большое (20+) | Для экстремальных структурных различий | Значительно медленнее; высокий риск ложных совпадений |

## Параметр смещения {#shift}

### Что он делает {#shift-what}

Параметр смещения сдвигает центр окна поиска. При `shift = 0` инструмент ожидает, что предложение `i` источника соответствует предложению `i` цели (соответствие один к одному). При `shift = 5` ожидается, что предложение `i` источника соответствует предложению `i + 5` цели (в целевом тексте примерно на 5 предложений больше перед текущей позицией).

### Зачем нужно смещение {#shift-why}

В идеальном мире параллельные тексты имели бы одинаковое число предложений. На практике так почти никогда не бывает:

- Переводчики добавляют пояснительные предложения
- Переводчики сокращают или объединяют предложения
- Разные правила разбиения на предложения дают разное число предложений
- Один текст может содержать вступительный материал (эпиграфы, посвящения), которого нет в другом

Когда число предложений различается, развивается прогрессивное смещение. Если цель содержит больше предложений, чем источник, каждое следующее предложение источника соответствует предложению цели с более высоким индексом, чем ожидалось. Без коррекции смещения диапазон поиска в итоге отстаёт от фактической позиции, и инструмент начинает выдавать плохие совпадения.

### Вычисление идеального смещения {#shift-computing}

Грубую оценку необходимого смещения можно вычислить из числа предложений:

```
estimated_shift ≈ (target_count - source_count) / num_batches
```

Например, если исходный текст содержит 1000 предложений, целевой — 1100, и вы используете 10 пакетов по 100 предложений, смещение на пакет составит примерно:

```
(1100 - 1000) / 10 = 10
```

Это означает, что к концу каждого пакета сдвиг вырос примерно на 10 предложений, поэтому смещение около 5–10 будет уместным (центрируя окно на ожидаемой средней точке сдвига).

На практике смещение редко бывает равномерным — одни разделы могут переводиться ближе к оригиналу, другие — свободнее. Режим автооптимизации Lingtrain может вычислить смещение для каждого пакета автоматически.

### Положительное и отрицательное смещение {#shift-direction}

- **Положительное смещение**: Целевой текст имеет больше предложений, чем источник. Инструмент ищет совпадения «впереди» в целевом тексте. Характерно, когда переводчик расширил текст или когда целевой язык склонен использовать больше предложений для того же содержания.
- **Отрицательное смещение**: Целевой текст имеет меньше предложений, чем источник. Инструмент ищет совпадения «позади» в целевом тексте. Характерно, когда переводчик сократил текст.
- **Нулевое смещение**: Источник и цель имеют приблизительно одинаковое число предложений. Значение по умолчанию для большинства хорошо совпадающих текстов.

## Примеры для разных соотношений текстов {#examples}

### Пример 1: Близкое совпадение (соотношение ~1:1) {#example-close}

- Источник: 500 предложений (английский)
- Цель: 510 предложений (французский)
- Разница: 10 предложений на 500 = 2% больше в цели

**Рекомендуемые настройки**: `window = 4`, `shift = 0`

Разница минимальна (2%). Окно по умолчанию 4 может поглотить этот небольшой сдвиг по пакетам. Корректировка смещения не нужна.

### Пример 2: Умеренная разница (соотношение ~1:1.15) {#example-moderate}

- Источник: 800 предложений (английский)
- Цель: 920 предложений (японский)
- Разница: 120 предложений на 800 = 15% больше в цели

**Рекомендуемые настройки**: `window = 6-8`, `shift = 2-3` (на пакет из 100)

Японские переводы английского текста часто дают больше предложений из-за другой структуры предложений. 15% разница требует умеренного увеличения окна и небольшого положительного смещения для удержания поиска в правильной области.

### Пример 3: Большая разница (соотношение ~1:1.3) {#example-large}

- Источник: 600 предложений (английский)
- Цель: 780 предложений (русский)
- Разница: 180 предложений на 600 = 30% больше в цели

**Рекомендуемые настройки**: `window = 10-12`, `shift = 4-6` (на пакет из 100)

30% разница значительна. Окно должно быть достаточно большим, чтобы охватить растущий сдвиг, а смещение должно компенсировать систематическую разницу. Рассмотрите использование меньших пакетов (50 вместо 100) для уменьшения внутрипакетного сдвига.

### Пример 4: Экстремальная разница (соотношение ~1:1.5+) {#example-extreme}

- Источник: 400 предложений (китайский)
- Цель: 650 предложений (английский)
- Разница: 250 предложений на 400 = 62% больше в цели

**Рекомендуемые настройки**: `window = 15-20`, `shift = 8-12` (на пакет из 50–80)

Китайский текст обычно значительно компактнее английского. Разница 60%+ требует агрессивной настройки параметров. Используйте меньшие пакеты (50–80 предложений), чтобы удержать внутрипакетный сдвиг в управляемых пределах, большое окно и значительное положительное смещение. Режим автооптимизации настоятельно рекомендуется для этого сценария.

## Режим автооптимизации {#auto-optimize}

Lingtrain предлагает режим автооптимизации, который автоматически вычисляет параметр смещения на основе соотношения числа предложений между исходным и целевым текстами. Этот режим:

1. Вычисляет общее соотношение числа предложений
2. Оценивает ожидаемое смещение на каждой границе пакета
3. Устанавливает параметр смещения для каждого пакета независимо, корректируя накопленный сдвиг

Автооптимизация рекомендуется для:
- Первого выравнивания (когда вы не знаете идеальных параметров)
- Текстов со значительной разницей в длине
- Текстов, где сдвиг варьируется по разделам (например, одни главы переведены близко к тексту, другие сильно перефразированы)

При необходимости вы можете вручную переопределить автоматически вычисленное смещение для отдельных пакетов.

## Распространённые сценарии, требующие ручной настройки {#common-scenarios}

### Сценарий: Глава с длинным примечанием переводчика {#scenario-note}

Переводчик добавил пояснительное примечание из 20 предложений в середину главы. Оставшиеся предложения смещены на 20 позиций. **Решение**: Увеличить смещение для пакетов после примечания или разбить выравнивание на границе примечания.

### Сценарий: Сокращённый перевод {#scenario-condensed}

Переводчик систематически сокращал текст, создав примерно 80% от числа предложений оригинала. **Решение**: Использовать отрицательное смещение (примерно -20% от размера пакета) и несколько большее окно.

### Сценарий: Разные издания {#scenario-editions}

Источник и цель — разные издания книги. Некоторые главы расположены в другом порядке. **Решение**: Это выходит за рамки того, что могут решить окно и смещение. Выравнивайте совпадающие главы по отдельности, а не всю книгу целиком.

### Сценарий: Поэзия с неравным числом строк {#scenario-poetry}

Поэтический перевод, где переводчик иногда использует больше или меньше строк на строфу. **Решение**: Используйте большее окно (10–15) для обработки вариативности. Рассмотрите выравнивание по строфам, а не по строкам.

### Сценарий: Смешанный контент (повествование + диалог) {#scenario-mixed}

Текст чередуется между длинными описательными фрагментами (переведёнными точно) и диалогом (переведённым свободно). **Решение**: Окно 6–8 справляется с большинством случаев. Если диалоговые участки систематически вызывают проблемы, рассмотрите их выравнивание отдельно с большим окном.

## Связь между окном, смещением и размером пакета {#relationship}

Эти три параметра взаимодействуют:

- **Большие пакеты** накапливают больше смещения внутри пакета, требуя больших окон и более точных значений смещения
- **Меньшие пакеты** имеют меньше внутрипакетного смещения, допуская меньшие окна и менее критичные значения смещения
- **Смещение действует на пакет**: Смещение 5 в пакете из 100 предложений компенсирует 5% разницу; то же смещение 5 в пакете из 50 компенсирует 10% разницу

Общее правило:
```
window ≥ abs(expected_offset_per_batch) + 3
```

Где `expected_offset_per_batch` — прогнозируемая разница числа предложений внутри одного пакета. `+ 3` обеспечивает запас на локальную вариативность.

## Устранение проблем {#troubleshooting}

| Симптом | Вероятная причина | Решение |
|---------|------------------|---------|
| Хорошее выравнивание в начале пакета, плохое в конце | Смещение слишком мало; сдвиг накапливается | Увеличить смещение или уменьшить размер пакета |
| Плохое выравнивание по всему пакету | Окно слишком мало для структурной разницы | Увеличить окно |
| Много ложных совпадений (неверные пары предложений) | Окно слишком велико; инструмент находит случайные совпадения | Уменьшить окно |
| Качество выравнивания сильно варьируется между пакетами | Неравномерная структура текста | Использовать автооптимизацию; рассмотреть настройку параметров для каждого пакета |
| Диаграмма рассеяния показывает диагональ с неверным наклоном | Систематическая разница числа предложений | Подобрать смещение, соответствующее наклону |
