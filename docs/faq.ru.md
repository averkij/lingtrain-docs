# Часто задаваемые вопросы {#faq}

На этой странице собраны ответы на самые распространённые вопросы о Lingtrain Aligner. Если вашего вопроса здесь нет, загляните в [Руководство по устранению неполадок](troubleshooting.ru.md) или воспользуйтесь формой обратной связи в приложении.

---

## Общие вопросы {#general}

### Что такое Lingtrain Aligner? {#what-is-lingtrain}

Lingtrain Aligner — это веб-приложение на основе машинного обучения для создания параллельных корпусов и двуязычных книг. Оно берёт два текста на разных языках (например, роман и его перевод) и автоматически выравнивает их на уровне предложений. Результат — структурированный набор пар предложений, который можно экспортировать в виде параллельных книг, баз переводческой памяти, корпусов и других форматов.

### Для кого предназначен Lingtrain? {#who-is-it-for}

Lingtrain рассчитан на широкую аудиторию:

- **Изучающие иностранные языки** — чтение двуязычных книг с параллельным переводом
- **Переводчики** — создание баз переводческой памяти (TMX) из существующих переводов
- **Исследователи NLP** — подготовка параллельных корпусов для обучения машинного перевода и лингвистического анализа
- **Преподаватели** — подготовка двуязычных учебных материалов
- **Издатели** — выпуск двуязычных изданий

### Lingtrain бесплатный? {#is-it-free}

Lingtrain предлагает бесплатный тариф для индивидуальных пользователей. Использование измеряется количеством обработанных строк. У каждого аккаунта есть квота, которая периодически обновляется. Платные тарифы доступны для больших объёмов и повышенных лимитов параллельных задач. Подробности — в настройках вашего аккаунта.

### Нужно ли что-то устанавливать? {#installation}

Нет. Lingtrain Aligner — это веб-приложение, которое целиком работает в браузере. Вам нужен только современный браузер (Chrome, Firefox, Edge или Safari) и подключение к интернету. Ничего скачивать и устанавливать не требуется.

## Форматы файлов и загрузка {#files}

### В каком формате должны быть тексты? {#file-format}

Lingtrain принимает **текстовые файлы** (`.txt`) в кодировке UTF-8. Форматы PDF, DOCX, EPUB и HTML напрямую не поддерживаются. Преобразуйте тексты в простой текстовый формат перед загрузкой.

### Каков максимальный размер файла? {#max-file-size}

Максимальный размер одного загружаемого документа по умолчанию — **4 МБ**. Этого достаточно для большинства полноразмерных романов. Если файл превышает этот лимит, попробуйте разделить его на части.

### Можно ли загрузить несколько файлов сразу? {#multiple-files}

Каждая область загрузки принимает по одному файлу за раз. Вы можете последовательно загрузить несколько документов для одного и того же языка. Каждый загруженный документ появляется в списке и может использоваться в разных выравниваниях.

### Какая кодировка нужна для текстовых файлов? {#encoding}

Файлы должны быть в кодировке **UTF-8**. Если ваш файл использует другую кодировку (например, Windows-1251 или ISO-8859-1), пересохраните его в UTF-8. Большинство текстовых редакторов (Notepad++, VS Code, Sublime Text) позволяют это сделать.

### Что делает опция «Очистить текст»? {#clean-text}

Флажок «Очистить текст» применяет базовую нормализацию при загрузке: удаление лишних пробелов, исправление артефактов кодировки и стандартизацию переводов строк. Включите её, если текст был извлечён из PDF или содержит проблемы форматирования.

## Языки {#languages}

### Сколько языков поддерживает Lingtrain? {#language-count}

Lingtrain явно поддерживает **22 языка** с отдельными правилами разбиения на предложения: русский, английский, немецкий, французский, испанский, итальянский, португальский, турецкий, польский, венгерский, чешский, китайский, японский, корейский, шведский, нидерландский, украинский, белорусский, башкирский, чувашский, армянский, а также режим «General» (универсальный).

Режим «General» позволяет работать с **200+ дополнительными языками**, поддерживаемыми моделями эмбеддингов (LaBSE, SONAR). Разбиение на предложения в этом режиме использует универсальный токенизатор, подходящий для большинства языков с латиницей и кириллицей.

### Можно ли выровнять тексты на языках, которых нет в списке? {#unlisted-languages}

Да. Выберите **«General»** для обоих языков. Модель эмбеддингов всё равно создаёт осмысленные векторы предложений для большинства языков, даже без специфических правил разбиения. Качество зависит от того, насколько хорошо данный язык представлен в обучающих данных модели.

### Для каких языковых пар результаты лучше всего? {#best-pairs}

Лучшие результаты дают пары с хорошей представленностью в модели эмбеддингов: английский-русский, английский-немецкий, английский-французский, английский-испанский, английский-китайский и большинство комбинаций крупных европейских языков. Для менее распространённых пар рассмотрите использование [подстрочника](proxy.ru.md).

### Поддерживает ли Lingtrain языки с написанием справа налево (арабский, иврит)? {#rtl-languages}

Модели эмбеддингов поддерживают арабский и иврит. Вы можете выравнивать тексты на этих языках, используя режим «General». Веб-интерфейс отображает текст согласно настройкам браузера, поэтому текст справа налево будет корректно показан в редакторе и экспорте.

## Процесс выравнивания {#alignment-process}

### Как работает алгоритм выравнивания? {#how-it-works}

Lingtrain преобразует каждое предложение в числовой вектор (эмбеддинг) с помощью многоязычной нейронной языковой модели. Затем вычисляет косинусное сходство между всеми эмбеддингами исходных и целевых предложений в пределах окна обработки и выбирает лучшее соответствие для каждого исходного предложения. Подробное объяснение — на странице [Алгоритм выравнивания](algorithm.ru.md).

### Что такое батчи? {#what-are-batches}

Большие тексты обрабатываются сегментами, которые называются батчами (пакетами). Каждый батч охватывает фиксированное число предложений исходного текста (по умолчанию 200) и пропорционально соответствующие предложения целевого текста. Батчи позволяют обрабатывать текст поэтапно, отслеживая прогресс и не перегружая память.

### Что такое параметр «окно» (window)? {#window-parameter}

Параметр «окно» определяет, сколько дополнительных предложений включается по обе стороны от пропорционального диапазона целевого текста для каждого батча. Большее окно (например, 60) увеличивает шанс найти правильные соответствия при значительном расхождении числа предложений, но также увеличивает время вычислений.

### Что такое параметр «сдвиг» (shift)? {#shift-parameter}

Параметр «сдвиг» вручную смещает позицию окна целевого текста. Используйте его, когда пропорция между количеством предложений в исходном и целевом текстах меняется по ходу текста. Например, если в целевом тексте в начальных главах значительно больше предложений, положительный сдвиг сместит окно вперёд для компенсации.

### Что такое конфликты? {#what-are-conflicts}

Конфликты — это места, где алгоритм не смог установить последовательное отображение «один к одному». Они возникают, когда переводчик разделил, объединил или переставил предложения. Конфликты обнаруживаются автоматически и могут быть разрешены с помощью встроенного алгоритма или вручную в редакторе.

### Как разрешить конфликты? {#resolving-conflicts}

Нажмите **«Resolve all»** (Разрешить все) в разделе конфликтов для запуска автоматического разрешения. Система перебирает все возможные группировки конфликтных предложений, вычисляет объединённые эмбеддинги для каждой группировки и выбирает вариант с наивысшим суммарным показателем сходства. После автоматического разрешения проверьте оставшиеся конфликты в редакторе.

### Можно ли остановить выравнивание, пока оно выполняется? {#stopping-alignment}

Да. Нажмите кнопку **«Stop»** (Остановить) во время выполнения выравнивания. Весь прогресс до текущего момента сохранится. Вы можете продолжить, нажав «Align next» снова.

### Что делает «Align all»? {#align-all}

«Align all» (Выровнять все) перезапускает обработку всех батчей с нуля, перезаписывая предыдущие результаты. Используйте эту функцию, когда хотите начать заново с другими настройками (например, другим сдвигом или окном). Обратите внимание, что при этом стираются все предыдущие данные выравнивания, включая ручные правки.

## Подстрочник (прокси-текст) {#proxy}

### Что такое подстрочник? {#what-is-proxy}

Подстрочник (прокси-текст) — это машинный перевод вашего разбитого на предложения документа, который используется как промежуточный текст при выравнивании. Вместо прямого сравнения оригинальных предложений (что может плохо работать для редких языковых пар) выравниватель сравнивает переводы-подстрочники, сохраняя оригинальные тексты в итоговом результате.

### Когда следует использовать подстрочник? {#when-proxy}

Используйте подстрочник, когда:

- Один или оба языка являются малоресурсными (например, башкирский, чувашский, миноритарные языки)
- Прямое выравнивание даёт плохое качество с большим количеством конфликтов
- Языковая пара очень дистантная (например, японский-финский)

Для хорошо поддерживаемых пар вроде английский-русский, английский-немецкий или английский-французский подстрочник обычно не нужен.

### Как создать подстрочник? {#create-proxy}

1. Загрузите и разбейте исходный текст обычным способом
2. Скачайте разбитый текст кнопкой «Скачать» в панели предпросмотра предложений
3. Переведите скачанный файл с помощью любого сервиса машинного перевода (Google Translate, DeepL и т. д.)
4. Загрузите переведённый файл как прокси-документ на странице деталей выравнивания

Подстрочник должен содержать **ровно столько же строк**, сколько в оригинальном разбитом тексте.

## Экспорт {#export}

### Какие форматы экспорта доступны? {#export-formats}

Lingtrain поддерживает следующие форматы экспорта:

| Формат | Расширение | Назначение |
|--------|------------|------------|
| HTML-книга | `.html` | Параллельная книга для чтения в браузере |
| TMX | `.tmx` | Переводческая память для CAT-инструментов |
| Корпус предложений | `.txt` | По одному предложению на строку для NLP-исследований |
| Корпус абзацев | `.txt` | Выровненный текст на уровне абзацев |
| XML | `.xml` | Структурированные данные для конвейеров обработки |
| JSON | `.json` | Структурированные данные для приложений |
| База выравнивания | `.lt` | Полная резервная копия для повторного импорта |

Подробные спецификации каждого формата см. в [Справочнике форматов экспорта](export-formats-reference.ru.md).

### Можно ли настроить внешний вид HTML-книги? {#customize-html}

Да. В настройках экспорта можно выбрать один из трёх стилей подсветки: **Simple** (без фона), **Pastel Fill** (сплошной пастельный фон) и **Pastel Start** (градиентный фон). Также можно задать направление абзацной структуры и порядок языков.

### Какие CAT-инструменты открывают TMX-файлы? {#tmx-cat-tools}

TMX (Translation Memory eXchange) — стандартный формат, поддерживаемый практически всеми CAT-инструментами: SDL Trados Studio, memoQ, Memsource, OmegaT, Wordfast, CafeTran и Smartcat.

### Можно ли экспортировать частично выровненные тексты? {#partial-export}

Да. Экспорт доступен, как только обработан хотя бы один батч. В экспорт попадут только выровненные части текста.

## Качество {#quality}

### Как оценить качество выравнивания? {#quality-assessment}

На странице деталей выравнивания отображается **chain score** (метрика качества) для каждого батча и общая. Значение 1.0 означает идеальное последовательное выравнивание. Значения выше 0.95 указывают на хорошее качество. Дополнительно **визуализация** (точечный график) должна показывать чистую диагональную линию.

### Что делать, если качество выравнивания плохое? {#poor-quality}

Несколько стратегий для улучшения качества:

1. **Подстройте сдвиг** — если диагональ смещена
2. **Увеличьте окно** — чтобы дать алгоритму больше пространства для поиска
3. **Используйте подстрочник** для малоресурсных языков
4. **Тщательнее подготовьте тексты** — удалите сноски, номера страниц, примечания переводчика
5. **Проверьте совпадение разметки** — оба текста должны иметь одинаковые структурные теги
6. **Попробуйте другую модель** эмбеддингов, если она доступна для вашей языковой пары

### Какую модель эмбеддингов выбрать? {#choosing-model}

Селектор моделей показывает доступные модели для вашей языковой пары. **LaBSE** рекомендуется для большинства пар, особенно с участием неевропейских языков. Для некоторых языковых комбинаций могут быть доступны специализированные модели. Если не уверены — модель по умолчанию подойдёт в большинстве случаев.

## Аккаунт и данные {#account}

### Как создать аккаунт? {#create-account}

Нажмите **«Sign in»** (Войти) на главной странице, затем выберите **«Create account»** (Создать аккаунт). Введите адрес электронной почты и пароль. Вам придёт код подтверждения по email. Введите код для активации аккаунта. Вы также можете войти через Google, Яндекс или ВКонтакте (OAuth).

### Мои данные приватны? {#data-privacy}

Да. Документы и выравнивания каждого пользователя изолированы. Другие пользователи не имеют доступа к вашим файлам.

### Могу ли я удалить свои данные? {#delete-data}

Вы можете удалить отдельные документы и выравнивания в любой момент через интерфейс. Все связанные файлы удаляются с сервера при удалении.

### Что делать, если я потерял выравнивание? {#backup}

Вы можете в любой момент экспортировать базу выравнивания (формат `.lt`) в качестве резервной копии. Этот файл содержит полное состояние выравнивания, включая разбиение на предложения, карту выравнивания и историю правок. Файлы `.lt` можно повторно импортировать для восстановления выравнивания.

## Доступ к API {#api-access}

### Есть ли у Lingtrain API? {#api}

Да. Lingtrain предоставляет REST API для всех операций. Вы можете загружать документы, создавать выравнивания, запускать обработку, разрешать конфликты и экспортировать результаты программно.

### Как пройти аутентификацию в API? {#api-auth}

API использует JWT-аутентификацию (JSON Web Token). Получите токен, отправив учётные данные на эндпоинт `/api/auth/login`. Добавляйте токен в заголовок `Authorization: Bearer <token>` последующих запросов.

## Распространённые ошибки {#common-errors}

### Ошибка «File too large» {#file-too-large}

Файл превышает максимальный размер загрузки (по умолчанию 4 МБ). Уменьшите размер файла, разделив его на части или удалив ненужное содержимое.

### Ошибка «File already exists» {#file-exists}

Файл с таким именем уже загружен для данного языка. Переименуйте файл или сначала удалите существующий.

### «Concurrent task limit reached» {#concurrent-limit}

Вы достигли максимального количества одновременных задач обработки, допускаемых вашим тарифом. Дождитесь завершения текущей задачи, прежде чем отправлять новую, или перейдите на тариф с более высокими лимитами.

### «Alignment uses an unsupported embedding model» {#unsupported-model}

Модель эмбеддингов, использованная при создании данного выравнивания, больше недоступна на сервере. Необходимо очистить данные выравнивания и выбрать доступную модель для повторного выравнивания.

### Визуализация показывает разбросанные точки вместо диагонали {#bad-visualization}

Это указывает на низкое качество выравнивания. Ознакомьтесь с [советами по улучшению качества](#poor-quality) выше или обратитесь к [Руководству по устранению неполадок](troubleshooting.ru.md).
