# Lingtrain для профессиональных переводчиков {#use-cases-translators}

Профессиональные переводчики работают с памятями переводов (TM) — базами данных ранее переведённых сегментов, которые помогают поддерживать единообразие и ускоряют будущие переводы. Lingtrain Aligner позволяет переводчикам создавать памяти переводов из существующих переведённых текстов, даже если эти переводы были выполнены без какого-либо CAT-инструмента.

На этой странице объясняется, как переводчики могут использовать Lingtrain для создания TM, интеграции с CAT-инструментами, извлечения терминологии и контроля качества переводов.

## Создание памятей переводов из существующих переводов {#building-tms}

### Проблема унаследованных переводов {#legacy-problem}

У каждого опытного переводчика есть корпус прошлых работ — книги, документы, статьи, — которые были переведены без CAT-инструмента. Эти переводы содержат ценные переводческие решения, последовательные терминологические выборы и предметно-специфические формулировки, наработанные годами.

Без выравнивания это знание заперто в неструктурированных текстовых файлах. Вы знаете, что где-то в вашем переводе 300-страничного технического руководства вы установили, как переводить «throttle body assembly», но чтобы найти это, нужно вручную искать по всему документу.

Lingtrain решает эту проблему, выравнивая исходный и целевой тексты на уровне предложений, создавая структурированную базу данных переводных пар, которую можно экспортировать как TMX-файл и импортировать в любой CAT-инструмент.

### Рабочий процесс создания TM {#tm-workflow}

1. **Соберите материалы.** Найдите исходный текст и ваш перевод в цифровом виде (текстовые файлы). Если у вас есть только печатные или PDF-версии, сначала потребуется OCR.

2. **Загрузите и выровняйте.** Загрузите оба текста в Lingtrain, создайте выравнивание и запустите пакетную обработку. Для профессиональных переводов качество выравнивания обычно высокое, поскольку профессиональные переводы точно следуют структуре оригинала.

3. **Тщательно проверьте.** Даже в профессиональных переводах есть разбиения и объединения предложений. Используйте разрешитель конфликтов, затем проверьте в редакторе. Для целей TM точность важнее, чем для обычного чтения — ошибочное выравнивание становится ошибочной записью TM.

4. **Экспортируйте в TMX.** Скачайте выравнивание в формате TMX. Этот файл можно импортировать в любой CAT-инструмент (SDL Trados, memoQ, OmegaT, Memsource, Smartcat и другие).

5. **Импортируйте и используйте.** Импортируйте TMX в ваш CAT-инструмент. Ваши прошлые переводы теперь доступны как память переводов, обеспечивая нечёткие и точные совпадения для будущих проектов.

### Советы по качеству TM {#tm-quality-tips}

- **Тщательно очистите исходные тексты.** Удалите номера страниц, колонтитулы, сноски и непереводимое содержание перед загрузкой. Эти элементы создают шум в TM.

- **Используйте теги разметки для структуры.** Пометьте заголовки глав тегом `%%%%%h1.` и другие структурные элементы. Они исключаются из выравнивания, предотвращая ложные соответствия.

- **Проверьте разбиение на предложения.** Перед запуском выравнивания просмотрите предпросмотр предложений на вкладке «Документы». Если разделитель некорректно определяет границы предложений, подумайте о корректировке текста.

- **Сосредоточьтесь на точности, а не полноте.** Лучше удалить несколько сомнительных пар в редакторе, чем включить неверные в TM. Неправильные совпадения TM хуже, чем их отсутствие.

## Интеграция с CAT-инструментами {#cat-integration}

### Совместимость формата TMX {#tmx-compatibility}

Lingtrain экспортирует TMX 1.4, поддерживаемый всеми основными CAT-инструментами:

| CAT-инструмент | Импорт TMX | Примечания |
|---------------|-----------|------------|
| SDL Trados Studio | Да | Импорт через Translation Memories > Import |
| memoQ | Да | Импорт через Resource Console > TMs |
| OmegaT | Да | Поместите TMX в папку /tm проекта |
| Memsource (Phrase) | Да | Импорт через Translation Memories |
| Smartcat | Да | Загрузка как ресурс TM |
| CafeTran | Да | Импорт через Memory > Import TMX |
| Wordfast | Да | Импорт через TM Management |

### Создание предметных TM {#domain-tms}

Переводчики, специализирующиеся в определённых областях (юридическая, медицинская, техническая, литературная), могут создавать специализированные TM из прошлых работ:

1. **Юридическая TM** — выравнивание контрактов, законодательства, судебных решений
2. **Медицинская TM** — выравнивание клинических исследований, информационных листков, медицинских статей
3. **Техническая TM** — выравнивание руководств пользователя, спецификаций, технической документации
4. **Литературная TM** — выравнивание романов, рассказов, переводов поэзии (полезно для серий, где терминология и имена должны оставаться единообразными)

Каждая предметная TM становится ценнее со временем по мере добавления новых выровненных переводов.

## Извлечение терминологии {#terminology}

### От выровненного корпуса к терминологической базе {#term-extraction}

Корпус, выровненный на уровне предложений, — отправная точка для извлечения терминологии:

1. **Экспортируйте** выравнивание как корпус предложений (параллельные текстовые файлы)
2. **Выявите повторяющиеся термины** — слова или фразы, которые последовательно переводятся одинаково в нескольких предложениях
3. **Создайте терминологическую базу** — составьте глоссарий пар «термин оригинала — термин перевода» с примерами контекста

Хотя Lingtrain не выполняет автоматическое извлечение терминологии, созданный им выровненный корпус — необходимый вход для терминологических инструментов и ручного составления глоссариев.

### Проверка единообразия {#consistency}

Качественно выровненный корпус позволяет проверить, переводили ли вы один и тот же термин единообразно:

- Найдите исходный термин в параллельном тексте и просмотрите все его переводы
- Выявите случаи, где один и тот же исходный термин переведён по-разному без веских оснований
- Обновите терминологическую базу, указав предпочтительные варианты перевода

## Контроль качества перевода {#quality-assurance}

### Проверка существующих переводов {#checking-translations}

Lingtrain может служить инструментом контроля качества для проверки полноты и точности перевода:

- **Пропущенное содержание:** если визуализация показывает большой разрыв (диагональ скачет), содержание могло быть случайно пропущено в переводе
- **Добавленное содержание:** лишние предложения в переводе, не соответствующие ни одному исходному, могут указывать на несанкционированные добавления переводчика
- **Структурные проблемы:** зашумлённая визуализация может означать, что перевод значительно перестроил оригинал, что может быть или не быть уместным

### Проверка работы другого переводчика {#reviewing}

При рецензировании чужого перевода:

1. Выровняйте исходный текст с переводом
2. Используйте редактор для просмотра выровненных пар предложений
3. Бок-о-бок представление позволяет легко заметить пропуски, добавления и ошибки перевода
4. Экспортируйте выравнивание для подробных замечаний

## Работа с литературными переводами {#literary}

### Серии и повторяющиеся персонажи {#series}

Для переводчиков, работающих над книжными сериями, единообразие между томами критически важно. Lingtrain помогает:

- Создать TM из первой книги перед началом работы над второй
- Обеспечить единообразие перевода имён персонажей, названий мест и повторяющихся фраз
- Создать справочный корпус с возможностью поиска во время перевода

### Несколько изданий и редакции {#revisions}

При редактировании перевода или сравнении изданий:

1. Выровняйте исходный оригинал с первым переводом
2. Выровняйте тот же оригинал с отредактированным переводом
3. Экспортируйте оба как корпуса предложений
4. Сравните два перевода предложение за предложением для выявления изменений

## Практические соображения {#practical}

### Подготовка текста для переводчиков {#text-prep}

Тексты переводчиков часто имеют специфические особенности:

- **Сноски и концевые сноски** — удалите или обработайте отдельно, так как они нарушают выравнивание предложений
- **Примечания переводчика** — удалите перед выравниванием или пометьте специальными тегами
- **Различная структура абзацев** — перевод может иметь другие разрывы абзацев, чем оригинал; Lingtrain обрабатывает это через параметры структуры абзацев при экспорте
- **Теги форматирования** — если в оригинале было форматирование (жирный, курсив), оно обычно теряется в текстовом виде; сосредоточьтесь на выравнивании содержания

### Временные затраты {#time-investment}

Создание TM из существующего перевода с помощью Lingtrain обычно занимает:

- **Короткий текст (10-50 страниц):** 15-30 минут, включая проверку
- **Средний текст (50-200 страниц):** 1-2 часа, включая проверку
- **Длинный текст (200+ страниц):** 2-4 часа, включая проверку

Это единовременные затраты, которые окупаются каждый раз, когда TM находит совпадения в будущих проектах.
