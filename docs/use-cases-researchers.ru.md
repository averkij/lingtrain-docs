# Lingtrain для исследователей NLP {#use-cases-researchers}

Параллельные корпуса — фундаментальный ресурс для компьютерной лингвистики и обработки естественного языка. Lingtrain Aligner предоставляет исследователям быстрый интерактивный конвейер для создания качественных корпусов, выровненных на уровне предложений, из необработанных двуязычных текстов, с экспортом в форматы, готовые для исследовательских рабочих процессов.

На этой странице описано, как исследователи могут использовать Lingtrain для машинного перевода, кросс-языкового NLP, переводоведения, цифровых гуманитарных наук и других задач.

## Создание обучающих данных для МП {#mt-training-data}

### Почему важны специализированные параллельные корпуса {#why-custom}

Стандартные параллельные корпуса (Europarl, корпус ООН, OPUS) охватывают множество языковых пар, но ограничены в предметной области, регистре и стиле. Парламентский корпус не поможет построить систему медицинского перевода; корпус субтитров не покроет юридическую терминологию.

Lingtrain позволяет исследователям создавать **предметно-специфические параллельные корпуса** из любых доступных переводов:

- Выравнивание двуязычных учебников для создания корпуса в образовательной области
- Выравнивание параллельных новостных статей для журналистского перевода
- Выравнивание художественных переводов для исследований литературного МП
- Выравнивание технической документации для специализированных систем МП

### Рабочий процесс для исследователей МП {#mt-workflow}

1. **Соберите** исходные и целевые тексты в вашей предметной области
2. **Загрузите** в Lingtrain и запустите выравнивание
3. **Проверьте и исправьте** с помощью разрешителя конфликтов и редактора — качество обучающих данных МП напрямую влияет на качество модели
4. **Экспортируйте** как корпус предложений (два параллельных текстовых файла) или TMX
5. **Постобработайте** — отфильтруйте по уверенности выравнивания, удалите дубликаты, очистите
6. **Обучите** модель МП на полученных параллельных данных

Экспорт корпуса предложений создаёт два файла с одним предложением на строку, выровненные по номерам строк — стандартный входной формат для инструментов OpenNMT, Marian NMT и fairseq.

## Кросс-языковое NLP {#cross-lingual}

### Проекция разметки {#annotation-projection}

Параллельные корпуса обеспечивают **проекцию разметки** — перенос лингвистических аннотаций с ресурсно богатого языка на бедный:

1. Разметьте ресурсно богатую сторону (например, английскую) тегами NER, POS или деревьями зависимостей с помощью существующих инструментов
2. Используйте сентенциальное выравнивание для проекции аннотаций на ресурсно бедную сторону
3. Обучите модели для бедного языка на спроецированных аннотациях

Выравнивание Lingtrain на уровне предложений обеспечивает соответствие, необходимое для этой проекции. Экспортируйте в формате JSON или XML для сохранения идентификаторов выравнивания вместе с текстом.

### Кросс-языковой перенос обучения {#transfer-learning}

Параллельные корпуса используются для оценки и улучшения кросс-языковых моделей:

- **Оценка** — проверка того, хорошо ли модель, обученная на языке A, работает на языке B, путём оценки на выровненных парах предложений
- **Дообучение** — использование выровненных пар для тонкой настройки мультиязычных моделей на конкретные языковые пары
- **Контрастивное обучение** — обучение моделей с использованием выровненных и невыровненных пар как положительных и отрицательных примеров

### Извлечение двуязычных словарей {#lexicon-extraction}

Из корпуса, выровненного на уровне предложений, исследователи могут извлекать двуязычные соответствия на уровне слов и фраз:

1. Создайте корпус, выровненный на уровне предложений, в Lingtrain
2. Экспортируйте как корпус предложений
3. Примените инструменты пословного выравнивания (eflomal, fast_align, SimAlign) для извлечения соответствий на уровне слов
4. Агрегируйте пословные соответствия для построения двуязычных словарей

Этот конвейер особенно ценен для малоресурсных языков, где существующие словари ограничены. См. [Работа с малоресурсными языками](low-resource-languages.ru.md).

## Переводоведение {#translation-studies}

### Количественный анализ перевода {#quantitative-analysis}

Параллельные корпуса обеспечивают исследование перевода на основе данных:

- **Анализ переводческих сдвигов** — измерение того, насколько переводчики отклоняются от структуры оригинала (разбиения, слияния, перестановки предложений)
- **Исследования эксплицитации** — количественная оценка того, как часто переводчики добавляют явную информацию, отсутствующую в оригинале
- **Анализ упрощения** — измерение того, используют ли переводы более простую лексику и более короткие предложения, чем оригиналы
- **Сравнение стилей переводчиков** — сопоставление нескольких переводов одного оригинала для выявления индивидуальных предпочтений переводчика

Метаданные выравнивания Lingtrain (идентификаторы строк, группировки, данные о конфликтах) предоставляют богатую информацию для этих анализов. База данных выравнивания (формат `.lt`) сохраняет полную историю выравнивания.

### Корпусное преподавание перевода {#translation-teaching}

Преподаватели перевода могут использовать Lingtrain для:

- Создания выровненных корпусов студенческих переводов для анализа ошибок
- Сравнения студенческих переводов с профессиональными
- Построения справочных корпусов для конкретных переводческих проблем (идиомы, культурные реалии, техническая терминология)

## Цифровые гуманитарные науки {#digital-humanities}

### Литературные параллельные корпуса {#literary-corpora}

Исследователи цифровых гуманитарных наук используют параллельные корпуса литературных текстов для:

- **Сравнительного литературоведения** — изучение того, как произведения адаптируются между языками и культурами
- **Истории перевода** — сравнение переводов разных эпох для отслеживания изменения переводческих норм
- **Стилистики** — анализ того, как литературный стиль сохраняется или трансформируется в переводе
- **Интертекстуальности** — отслеживание кросс-языковых отсылок и влияний

Поддержка тегов разметки в Lingtrain (`%%%%%h1.`, `%%%%%author.`) сохраняет структурную информацию (главы, разделы) в выровненном корпусе, обеспечивая структурный анализ.

### Документирование исчезающих языков {#endangered-languages}

Для языков с ограниченными цифровыми ресурсами выравнивание существующих двуязычных текстов (фольклорные сборники, религиозные тексты, учебные материалы) создаёт структурированные параллельные данные для:

- Построения базовых инструментов NLP (токенизаторы, POS-теггеры)
- Обучения систем машинного перевода
- Создания двуязычных учебных материалов
- Сохранения переводческих знаний

## Форматы экспорта для исследований {#export-formats}

Lingtrain поддерживает несколько форматов экспорта для различных исследовательских задач:

### Корпус предложений {#sentence-corpus}

Два текстовых файла с выровненными предложениями, по одному на строку. Строки соответствуют друг другу по номерам.

**Применение:** обучение МП, статистический анализ, любой инструмент, ожидающий параллельный текст с построчным выравниванием.

### TMX (Translation Memory eXchange) {#tmx}

Стандартный XML-формат для памятей переводов. Каждая единица перевода содержит исходный и целевой сегменты с языковыми тегами.

**Применение:** интеграция с CAT-инструментами, переводоведение, любой конвейер обработки на основе XML.

### JSON и XML {#json-xml}

Структурированные форматы, сохраняющие метаданные выравнивания (идентификаторы, соответствия строк, группировки).

**Применение:** пользовательские конвейеры обработки, веб-приложения, импорт в базы данных, детальный анализ.

### База данных выравнивания (.lt) {#lt-format}

Собственный формат Lingtrain на основе SQLite, содержащий полные данные выравнивания: тексты предложений, эмбеддинги, соответствия, историю конфликтов и метаданные.

**Применение:** резервное копирование, повторный импорт в Lingtrain, прямые SQL-запросы для исследовательского анализа.

## Интеграция с Python {#python-integration}

Библиотека `lingtrain-aligner` для Python позволяет исследователям использовать алгоритм выравнивания программно:

- Интеграция выравнивания в конвейеры обработки данных
- Пакетная обработка нескольких пар текстов без веб-интерфейса
- Доступ к низкоуровневым данным выравнивания (матрицы сходства, детали конфликтов)
- Настройка параметров выравнивания для конкретных исследовательских задач

Документацию см. в [библиотека lingtrain-aligner для Python](python-library.ru.md).

## Советы для исследовательского использования {#tips}

1. **Отдавайте приоритет качеству выравнивания, а не скорости.** Для исследовательских корпусов потратьте время на проверку конфликтов и исправление ошибок. Зашумлённые выравнивания распространяют ошибки в последующие модели.

2. **Документируйте процесс.** Записывайте, какую модель, параметры и шаги постобработки вы использовали. Это обеспечивает воспроизводимость.

3. **Экспортируйте базу данных выравнивания.** Файл `.lt` сохраняет всё — вы всегда можете повторно экспортировать в другие форматы или импортировать для дальнейшего редактирования.

4. **Используйте визуализацию для оценки качества.** Точечные диаграммы обеспечивают быструю визуальную проверку качества выравнивания по всему тексту, выделяя проблемные области.

5. **Рассмотрите несколько моделей.** Для исследований может быть полезно выровнять одну и ту же пару текстов разными моделями эмбеддингов и сравнить результаты. Создайте отдельные выравнивания для каждой модели.
