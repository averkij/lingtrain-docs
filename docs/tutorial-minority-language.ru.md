# Пользовательская история: сохранение миноритарного языка через параллельные тексты {#minority-language}

Айдар — лингвист и культурный активист из Башкортостана. Он увлечён сохранением башкирского языка — тюркского языка, на котором говорят около 1,2 миллиона человек. Хотя башкирский имеет официальный статус в Республике Башкортостан, цифровые ресурсы для этого языка крайне скудны. Айдар хочет создать коллекцию параллельных башкирско-русских текстов, которые послужат учебными материалами для изучающих язык и цифровым ресурсом для исследований в области компьютерной лингвистики.

## Проблема {#problem}

Миноритарные языки сталкиваются с цифровым разрывом. В то время как для основных языков существуют огромные онлайн-корпусы, памяти переводов и коллекции параллельных текстов, для таких языков, как башкирский, ресурсов крайне мало:

- Мало оцифрованных литературных произведений.
- Практически отсутствуют параллельные корпусы.
- Ограниченная поддержка в инструментах NLP и языковых моделях.
- Растущая обеспокоенность утратой языка среди молодого поколения.

Айдар убеждён, что создание доступных параллельных текстов может помочь на нескольких фронтах: изучающие язык смогут использовать их для занятий, исследователи — для улучшения NLP-инструментов, а сами тексты станут формой цифровой сохранности.

## Поиск исходных материалов {#finding-materials}

Айдар находит несколько текстов, существующих и на башкирском, и на русском:

1. **Рассказы Мустая Карима** — прославленного башкирского писателя, чьи произведения широко переведены на русский. Некоторые находятся в общественном достоянии.
2. **Башкирские народные сказки** — традиционные истории, изданные в двуязычных сборниках.
3. **Государственные и образовательные материалы** — официальные документы, существующие на обоих языках по закону.

Он начинает со сборника коротких башкирских народных сказок, изданного в двуязычном башкирско-русском варианте. У него есть доступ к обоим текстам в цифровой форме.

## Трудность с прямым выравниванием {#proxy-challenge}

Башкирский — малоресурсный язык для большинства моделей эмбеддингов. Когда Айдар пробует прямое выравнивание (башкирский как исходный, русский как целевой), визуализация показывает хаотичную картину — модель не может надёжно сопоставить башкирские предложения с их русскими переводами.

Именно здесь выравнивание с подстрочником становится незаменимым.

## Создание подстрочника {#creating-proxy}

Подход Айдара:

1. **Загрузить башкирский текст** на вкладку Documents и проверить разбиение на предложения. Башкирский текст разбивается на 186 предложений.
2. **Скачать разбитый башкирский текст** — по одному предложению на строку.
3. **Перевести на русский** с помощью Яндекс.Переводчика, который достаточно хорошо поддерживает башкирский как тюркский язык. Яндекс — лучший выбор, чем Google Translate, для этой пары благодаря сильной поддержке языков бывшего Советского Союза.
4. **Проверить количество строк** — в подстрочнике ровно 186 строк, как в оригинале.
5. **Выборочная проверка качества** — Айдар читает по-башкирски как носитель и проверяет случайную выборку из 20 переведённых предложений. Примерно 15 хорошие, 3 приемлемые, 2 плохие (очень короткие предложения, переведённые некорректно). Общее качество достаточно для целей выравнивания.

Он загружает подстрочник на стороне «from» (башкирский) страницы деталей выравнивания.

## Запуск выравнивания {#running}

С включённым подстрочником Айдар запускает выравнивание:

1. Система вычисляет эмбеддинги из русского подстрочника (вместо оригинального башкирского текста) и сравнивает их с русским целевым текстом.
2. Поскольку обе стороны теперь на русском, модель эмбеддингов работает хорошо.
3. Визуализация показывает чистую диагональ — разительное улучшение по сравнению с прямой попыткой.

Система находит 12 конфликтов, и автоматическое разрешение справляется со всеми. Айдар просматривает результат в редакторе с включённым «Show subscriptions», чтобы видеть подстрочные переводы под каждым башкирским предложением. Пары выглядят корректно.

## Проверка качества {#quality-review}

Поскольку Айдар читает и по-башкирски, и по-русски, он может проверить каждую пару напрямую. Он проходит через 186 пар и обнаруживает:

- 178 пар выровнены идеально.
- У 5 пар есть незначительные проблемы (граница предложения чуть смещена), но они семантически корректны.
- 3 пары требуют ручной коррекции — два случая, когда короткая башкирская пословица была сопоставлена с неправильным русским предложением.

Он исправляет 3 проблемные пары с помощью функции Candidates в редакторе и доволен результатом.

## Экспорт и распространение {#exporting}

Айдар экспортирует выравнивание в нескольких форматах:

- **HTML-книга** — красивая параллельная башкирско-русская книга, которую изучающие язык могут читать в браузере. Он выбирает стиль Pastel для визуального разделения языков.
- **Корпус предложений** — для исследователей компьютерной лингвистики, которые хотят обучать или оценивать модели на этой языковой паре.
- **TMX** — для переводчиков, работающих с башкирским, которые хотят накапливать память переводов.
- **База данных выравнивания (.lt)** — для архивирования и для тех, кто захочет проверить или расширить выравнивание.

Он публикует HTML-книгу на сайте по изучению башкирского языка, который ведёт, и загружает корпусы в лингвистический репозиторий данных.

## Масштабирование проекта {#scaling}

Воодушевлённый успехом, Айдар расширяет проект:

1. Обрабатывает ещё сказки — ещё 500 пар предложений.
2. Добавляет рассказы Мустая Карима — примерно 2 000 пар предложений.
3. Привлекает двух добровольцев (учителя башкирского и студента университета) для помощи с проверкой качества.

За шесть месяцев проект создаёт параллельный корпус из приблизительно 4 000 башкирско-русских пар предложений — один из крупнейших открытых ресурсов для этой языковой пары.

## Влияние {#impact}

Параллельные тексты оказывают разнообразное влияние:

- **Изучающие язык** используют HTML-книги для изучения башкирской лексики и структуры предложений в контексте.
- **Университетская исследовательская группа** использует корпус предложений для дообучения мультиязычной модели для башкирского, улучшая её производительность на последующих задачах.
- **Местные переводчики** импортируют TMX в свои CAT-инструменты, помогая поддерживать единообразную терминологию.
- **Сообщество Башкирской Википедии** использует выровненные тексты как справочный материал для переводческой работы.

## Трудности и уроки {#challenges}

Айдар размышляет о трудностях, с которыми столкнулся:

1. **Качество подстрочника непостоянно.** Яндекс.Переводчик достаточно хорошо справляется с башкирским для обычной лексики, но затрудняется с архаичными или диалектными формами в народных сказках. Для таких случаев он иногда создаёт подстрочник вручную — медленнее, но точнее.
2. **Разбиение на предложения для башкирского несовершенно.** Опция «General» работает, но иногда пропускает границы предложений, специфичные для башкирских пунктуационных конвенций. Он предварительно обрабатывает тексты для обеспечения чистых разрывов абзацев.
3. **Культурный контекст важен.** Некоторые башкирские выражения не имеют прямого русского эквивалента. Выравнивание корректно сопоставляет предложения, но переводческое соотношение — скорее адаптация, чем дословный перевод.
4. **Участие сообщества — ключ.** Проверка качества носителями языка необходима для создания надёжного ресурса.
5. **Начинайте с малого, растите органично.** Начать со 186 предложений было правильным подходом — это позволило проверить рабочий процесс перед обработкой больших текстов.

## Рекомендации для других языковых активистов {#guidance}

На основе своего опыта Айдар даёт советы людям, работающим с другими миноритарными языками:

1. **Используйте выравнивание с подстрочником.** Это самая важная функция для малоресурсных языков. Без него прямое выравнивание может быть непригодным.
2. **Выбирайте лучший сервис перевода для вашего языка.** Протестируйте несколько. Для тюркских языков Яндекс часто превосходит Google. Для африканских языков Google может быть лучше. Проверяйте.
3. **Проверяйте качество подстрочника перед обработкой.** Плохой подстрочник даст плохое выравнивание. Выборочно проверьте не менее 10% переведённых предложений.
4. **Экспортируйте в нескольких форматах.** Разным аудиториям нужны разные форматы. Учащимся — HTML-книги, исследователям — корпусы, переводчикам — TMX.
5. **Делитесь открыто.** Ценность параллельных текстов умножается, когда они свободно доступны. По возможности публикуйте под разрешительной лицензией.
6. **Документируйте методологию.** Будущим участникам нужно знать, как был создан корпус, чтобы последовательно его расширять.
