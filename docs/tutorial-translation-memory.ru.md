# Пользовательская история: создание памяти переводов из существующих переводов {#translation-memory}

Елена — профессиональный переводчик с 15-летним стажем. За эти годы она перевела десятки технических руководств, маркетинговых материалов и художественных текстов с английского на русский. Большинство этих переводов существуют только как готовые документы — файлы Word, PDF и печатные книги. Она хочет преобразовать свои прошлые работы в переиспользуемую память переводов (TMX), которую можно загрузить в CAT-инструмент (memoQ) для ускорения будущих проектов.

## Проблема {#challenge}

Ситуация Елены типична для опытных переводчиков:

- У неё есть **оригинальные английские тексты** и их **русские переводы** как отдельные документы.
- Тексты переводились вручную, без CAT-инструмента — памяти переводов не существует.
- Её текущий CAT-инструмент (memoQ) может импортировать TMX-файлы, но создавать их вручную заняло бы месяцы.
- Она хочет переиспользовать прошлые переводы для единообразия терминологии и повторного использования в похожих проектах.

Lingtrain Aligner может решить эту задачу: загрузить оба текста, выровнять их на уровне предложений и экспортировать как TMX.

## Выбор текстов для обработки {#selecting}

Елена начинает с самых ценных переводов — тех, которые с наибольшей вероятностью содержат переиспользуемый контент:

1. **Руководство пользователя ПО** (120 страниц, ~30 000 слов) — высокоповторяемый технический контент с единообразной терминологией. Отличный потенциал повторного использования.
2. **Корпоративный годовой отчёт** (40 страниц, ~15 000 слов) — деловой язык со стандартными фразами.
3. **Серия продуктовых брошюр** (5 брошюр, ~5 000 слов каждая) — маркетинговый язык с повторяющимися описаниями продуктов.

Она решает начать с руководства ПО как пилотного проекта.

## Извлечение и подготовка текста {#preparation}

Оригинальное английское руководство — PDF, а её русский перевод — файл DOCX. Из обоих нужен обычный текст.

**Английский (PDF):**
Елена использует `pdftotext` для извлечения содержимого. В результате есть проблемы:
- Номера страниц на каждой странице.
- Колонтитулы ("Product Name — User Manual"), повторяющиеся повсюду.
- Переносы слов на разрывах строк ("config-\nuration" вместо "configuration").
- Содержимое таблиц, перемешанное с основным текстом.

Она очищает текст в редакторе:
- Удаляет все номера страниц и колонтитулы.
- Соединяет слова с переносами через разрыв строки.
- Удаляет содержимое таблиц (оно плохо выравнивается на уровне предложений).
- Убеждается, что абзацы разделены пустыми строками.

**Русский (DOCX):**
Она открывает DOCX в текстовом процессоре и сохраняет как обычный текст (UTF-8). Результат чище, но всё же требует:
- Удаления оглавления.
- Удаления подписей к рисункам, которых нет в английской версии.
- Объединения жёстко перенесённых строк.

Она добавляет минимальную разметку в оба файла:
```
Product Name%%%%%title.
User Manual%%%%%title.
```

И теги заголовков для основных разделов:
```
Chapter 1: Installation%%%%%h1.
Chapter 2: Configuration%%%%%h1.
```

## Загрузка и выравнивание {#aligning}

Елена загружает оба текста в Lingtrain. Английский разбивается на 1 247 предложений, русский — на 1 312 — разница в 5%, что типично для технического перевода, где русский текст обычно немного более многословен.

Она создаёт выравнивание и обрабатывает все пакеты. Визуализация показывает чистую диагональ — технические тексты с параллельной структурой выравниваются очень хорошо.

Система находит 42 конфликта. Она запускает «Resolve all», и счётчик падает до 3. Эти три оставшихся конфликта находятся в разделе, где русский перевод добавил пояснительные примечания, отсутствующие в английском. Елена открывает их в редакторе и удаляет лишние русские предложения — это примечания переводчика, не часть оригинального контента, и они не должны попадать в память переводов.

## Проверка качества {#reviewing}

Для памяти переводов качество первостепенно. Неправильная пара в TM будет всплывать как неверное предложение в будущих проектах, потенциально привнося ошибки.

Елена тщательно проверяет выравнивание:

1. Просматривает редактор по 20 строк на странице, проверяя каждую пару.
2. Обращает особое внимание на:
   - **Единообразие терминологии** — используют ли пары те же термины, которые она хотела бы переиспользовать?
   - **Границы предложений** — является ли каждая пара чистой, самодостаточной единицей перевода?
   - **Пустые ячейки** — нет ли пробелов, указывающих на отсутствующий контент?
3. Находит и исправляет 8 проблем:
   - 3 пары, где разбиение разрезало предложение посередине технического термина.
   - 2 пары, где маркированный список был ошибочно объединён в одно предложение.
   - 3 пары, где русский текст включал фрагмент из другого раздела.

Проверка занимает около 45 минут для выравнивания из 1 247 пар.

## Экспорт в TMX {#exporting}

Удовлетворённая качеством, Елена переходит на вкладку Create и скачивает TMX-файл.

TMX-файл содержит:

```xml
<tmx version="1.4">
  <header srclang="en" adminlang="en" datatype="plaintext"/>
  <body>
    <tu>
      <tuv xml:lang="en">
        <seg>Click the Install button to begin the installation process.</seg>
      </tuv>
      <tuv xml:lang="ru">
        <seg>Нажмите кнопку «Установить», чтобы начать процесс установки.</seg>
      </tuv>
    </tu>
    <!-- ... ещё единицы перевода ... -->
  </body>
</tmx>
```

## Импорт в memoQ {#importing}

Елена импортирует TMX-файл в memoQ:

1. Открывает свой проект memoQ.
2. Переходит в Translation memories > Import.
3. Выбирает TMX-файл.
4. memoQ импортирует все 1 247 единиц перевода.

Она сразу тестирует память на новом разделе похожего руководства ПО. Результаты впечатляют — memoQ показывает нечёткие совпадения для примерно 30% новых предложений, со многими точными совпадениями для стандартных фраз интерфейса, таких как "Click OK to confirm" / «Нажмите ОК для подтверждения».

## Обработка оставшихся текстов {#remaining}

Воодушевлённая результатами, Елена обрабатывает годовой отчёт и серию брошюр. Каждый следует тому же рабочему процессу:

1. Извлечь текст из оригинального формата.
2. Очистить и добавить теги разметки.
3. Загрузить, выровнять, разрешить, проверить.
4. Экспортировать как TMX.
5. Импортировать в memoQ.

Весь процесс для всех пяти документов занимает один полный рабочий день. Результат: память переводов с приблизительно 4 500 качественными единицами перевода, созданная из 15 лет работы.

## Долгосрочная ценность {#value}

В течение следующих шести месяцев Елена отслеживает эффект:

- **Процент повторного использования** в новых технических переводах увеличивается на 15-25%.
- **Единообразие терминологии** заметно улучшается — TM предлагает те же термины, которые она использовала ранее.
- **Скорость перевода** возрастает, так как она тратит меньше времени на ручной поиск прошлых переводов.
- **Удовлетворённость клиентов** повышается, так как они замечают более единообразную терминологию между проектами.

Она также делится частью TM с коллегой, работающей над похожим техническим контентом, дополнительно увеличивая ценность.

## Советы по созданию памяти переводов {#tips}

1. **Начинайте с самого повторяемого контента.** Технические руководства, документация к продуктам и юридические тексты имеют наибольший потенциал повторного использования.
2. **Тщательно очищайте тексты.** TMX с зашумлёнными парами хуже, чем полное отсутствие TMX — он привносит ошибки в будущие переводы.
3. **Проверяйте каждую пару.** Для качества TM ручная проверка обязательна. Заложите на это время.
4. **Удаляйте примечания и дополнения переводчика.** Включайте только контент, присутствующий и в оригинале, и в переводе.
5. **Экспортируйте резервные копии .lt.** Вам может понадобиться переэкспортировать с другими параметрами или внести исправления позже.
6. **Обрабатывайте партиями.** Не пытайтесь обработать все прошлые переводы за один раз. Начните с самых ценных текстов, проверьте процесс, затем продолжайте.
7. **Организуйте свою TM.** В CAT-инструменте создайте отдельные TM для разных доменов (технический, маркетинговый, юридический) для улучшения релевантности совпадений.
