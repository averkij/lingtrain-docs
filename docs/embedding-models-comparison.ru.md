# Сравнение моделей эмбеддингов {#embedding-models-comparison}

Lingtrain Aligner поддерживает несколько моделей эмбеддингов для вычисления сходства предложений между языками. Каждая модель имеет свои преимущества, языковое покрытие и характеристики производительности. Эта страница поможет вам выбрать подходящую модель для задачи выравнивания.

## Обзор {#overview}

Модель эмбеддингов определяет, как предложения преобразуются в векторы для сравнения. Выбор модели влияет на качество выравнивания, скорость обработки и языковое покрытие. Модель выбирается в панели **Управление** на странице деталей выравнивания перед запуском первого пакета.

![Панель управления с выбором модели](assets/screenshots/aligner-controls-panel.png)

## Доступные модели {#available-models}

### BGE-M3 {#bge-m3}

| Свойство | Значение |
|----------|----------|
| **Полное название** | BAAI General Embedding M3 |
| **Разработчик** | BAAI (Beijing Academy of Artificial Intelligence) |
| **Архитектура** | На основе XLM-RoBERTa, дообученная для поиска |
| **Размерность** | 1024 |
| **Языки** | 100+ |
| **Доступ** | Через API (OpenRouter) |

**BGE-M3** — универсальная модель эмбеддингов, разработанная для мультиязычного, мультифункционального поиска на различных уровнях гранулярности. Поддерживает плотный поиск, разрежённый поиск и мультивекторный поиск в одной модели.

**Достоинства:**

- Отличное качество для основных европейских и азиатских языков
- Высокие результаты на бенчмарках поиска (MTEB)
- Хорошо работает как с короткими предложениями, так и с длинными фрагментами
- Хороший баланс качества и скорости

**Недостатки:**

- Снижение качества для малоресурсных языков, слабо представленных в обучающих данных
- Требует доступа к API (недоступна для полностью автономного использования)

**Лучше всего подходит для:** Общего выравнивания основных мировых языков. Надёжный выбор по умолчанию при отсутствии специфических требований.

### LaBSE (через HuggingFace API) {#labse}

| Свойство | Значение |
|----------|----------|
| **Полное название** | Language-Agnostic BERT Sentence Embeddings |
| **Разработчик** | Google Research |
| **Архитектура** | Двойной кодировщик BERT |
| **Размерность** | 768 |
| **Языки** | 109 |
| **Доступ** | Через HuggingFace Inference API |

**LaBSE** была специально разработана для кросс-языкового поиска и сравнения предложений. Обучена на двуязычных парах предложений, извлечённых из веба, что обеспечивает широкое и надёжное языковое покрытие.

**Достоинства:**

- Целенаправленно создана для межъязыкового сходства предложений
- Хорошее покрытие среднересурсных и некоторых малоресурсных языков
- Обучена на извлечённых параллельных данных (не только на Википедии), обеспечивая более разнообразное языковое представление
- Хорошо проверена на задачах выравнивания и извлечения параллельных текстов

**Недостатки:**

- Более старая архитектура по сравнению с новыми моделями
- 768-мерные эмбеддинги могут улавливать меньше нюансов, чем более крупные модели
- Требует доступа к API HuggingFace

**Лучше всего подходит для:** Языковых пар с участием среднересурсных или малоресурсных языков. Особенно рекомендуется для армянского, башкирского и других языков, которые более крупные универсальные модели могут обслуживать недостаточно.

### Multilingual E5 Large {#e5-large}

| Свойство | Значение |
|----------|----------|
| **Полное название** | Multilingual E5 Large |
| **Разработчик** | Microsoft |
| **Архитектура** | XLM-RoBERTa Large, дообученная контрастивным обучением |
| **Размерность** | 1024 |
| **Языки** | 100+ |
| **Доступ** | Через API |

**Multilingual E5 Large** — высококачественная мультиязычная модель эмбеддингов от Microsoft, обученная по сложному многоэтапному процессу, включающему контрастивное предобучение на большом объёме текстовых пар и дообучение на размеченных данных.

**Достоинства:**

- Одни из лучших доступных мультиязычных эмбеддингов
- Отличные результаты на бенчмарках семантического сходства
- Сильные кросс-языковые возможности благодаря базе XLM-R
- 1024 измерения улавливают тонкие семантические различия

**Недостатки:**

- Более крупная модель означает более медленный вывод
- Может быть избыточна для простых задач выравнивания с высокоресурсными языками
- Требует доступа к API

**Лучше всего подходит для:** Случаев, когда нужны эмбеддинги наивысшего качества для сложных выравниваний — тексты со значительными структурными различиями, художественные произведения с творческими переводами или специализированный контент.

### OpenAI Text Embedding 3 Small {#openai}

| Свойство | Значение |
|----------|----------|
| **Полное название** | text-embedding-3-small |
| **Разработчик** | OpenAI |
| **Архитектура** | Проприетарный трансформер |
| **Размерность** | 1536 |
| **Языки** | 100+ (точное покрытие не опубликовано) |
| **Доступ** | Через OpenAI API (требуется API-ключ) |

**text-embedding-3-small** от OpenAI — коммерческая модель эмбеддингов с хорошими мультиязычными возможностями и высокой размерностью.

**Достоинства:**

- Качественные эмбеддинги с 1536 измерениями
- Хорошее мультиязычное покрытие для основных языков
- Быстрые ответы API
- Хорошо поддерживается и регулярно обновляется

**Недостатки:**

- Коммерческий API — оплата за каждый запрос
- Проприетарная модель — детали не полностью опубликованы
- Прозрачность покрытия языков недостаточна (сложно узнать, поддерживается ли конкретный малоресурсный язык)
- Требует доступа к сети и API-ключа

**Лучше всего подходит для:** Пользователей, у которых уже есть API-ключ OpenAI и которые хотят качественные эмбеддинги без забот о хостинге модели. Хороша для продуктивных рабочих процессов, где важна надёжность API.

### Qwen3 Embedding 4B {#qwen3}

| Свойство | Значение |
|----------|----------|
| **Полное название** | Qwen3 Embedding 4B |
| **Разработчик** | Alibaba Cloud (команда Qwen) |
| **Архитектура** | Трансформер Qwen3, 4 миллиарда параметров |
| **Размерность** | Переменная (зависит от модели) |
| **Языки** | 100+ с сильной поддержкой CJK |
| **Доступ** | Через API |

**Qwen3 Embedding 4B** — большая модель эмбеддингов из семейства Qwen от Alibaba, обладающая высокой ёмкостью для улавливания семантических нюансов.

**Достоинства:**

- Очень крупная модель (4 млрд параметров) улавливает тонкие семантические различия
- Отличное качество для китайского, японского, корейского наряду с европейскими языками
- Высокие результаты на кросс-языковых бенчмарках
- Хорошая обработка предметно-специфического и технического контента

**Недостатки:**

- Большой размер модели означает более медленный вывод
- Более высокая вычислительная стоимость на запрос
- Относительно новая модель, менее проверенная сообществом на задачах выравнивания

**Лучше всего подходит для:** Выравнивания с участием китайских, японских или корейских текстов, а также высокоспециализированного/технического контента, где важно улавливание тонких семантических различий.

## Как выбрать {#how-to-choose}

### Схема принятия решений {#flowchart}

1. **Вы работаете с китайским, японским или корейским?**
   - Да: Начните с **Qwen3 Embedding 4B** или **BGE-M3**
   - Нет: Переходите к шагу 2

2. **Вы работаете с малоресурсными или миноритарными языками?**
   - Да: Начните с **LaBSE**
   - Нет: Переходите к шагу 3

3. **Вам нужно максимально высокое качество?**
   - Да: Используйте **Multilingual E5 Large**
   - Нет: Переходите к шагу 4

4. **У вас есть API-ключ OpenAI и вы предпочитаете управляемый сервис?**
   - Да: Используйте **OpenAI text-embedding-3-small**
   - Нет: Используйте **BGE-M3** по умолчанию

### Практические советы {#tips}

- **Начните с модели по умолчанию.** BGE-M3 хорошо работает для большинства языковых пар. Меняйте только при проблемах с качеством на визуализации (разбросанные точки вместо чистой диагонали).

- **Попробуйте LaBSE для редких языков.** Если выравнивание включает башкирский, чувашский, армянский или коми, обучение LaBSE на извлечённых параллельных данных даёт ей преимущество.

- **Проверяйте визуализацию после первого пакета.** Точечная диаграмма — лучший индикатор качества эмбеддингов. Чистая диагональ означает, что модель хорошо работает для вашей языковой пары; разбросанные точки — нужно попробовать другую модель.

- **Рассмотрите подстрочник как дополнение.** Если ни одна модель не даёт хорошего прямого выравнивания для вашей языковой пары, попробуйте тексты-подстрочники (машинные переводы в качестве посредников). См. [Работа с малоресурсными языками](low-resource-languages.ru.md).

- **Выбор модели фиксируется для выравнивания.** Обработав первый пакет с выбранной моделью, эта модель используется для всех последующих пакетов того же выравнивания. Для пробы другой модели создайте новое выравнивание.

## Технические детали {#technical-details}

### Как настраиваются модели {#configuration}

Модели эмбеддингов управляются через панель администратора. Каждая модель имеет **ключ** (внутренний идентификатор для маршрутизации API) и **публичное название** (отображается в интерфейсе). Администраторы могут добавлять, редактировать и удалять модели по мере необходимости.

Ключ модели обычно имеет формат `api:provider:model-name`, например:

- `api:openrouter:baai/bge-m3`
- `api:huggingface:sentence-transformers/LaBSE`
- `api:openai:text-embedding-3-small`

### Процесс вычисления эмбеддингов {#computation-flow}

1. Фронтенд отправляет ключ выбранной модели с запросом на выравнивание
2. Бэкенд разрешает ключ модели в конечную точку API и конфигурацию
3. Предложения отправляются в API эмбеддингов пакетами (обычно по 10 предложений)
4. Полученные эмбеддинги L2-нормализуются и сохраняются в базе данных выравнивания
5. Матрица сходства вычисляется как скалярное произведение нормализованных матриц эмбеддингов

### Требования согласованности {#consistency}

Все предложения в выравнивании должны использовать одну и ту же модель эмбеддингов. Смешивание моделей сделало бы оценки сходства несопоставимыми, поскольку разные модели отображают в разные векторные пространства. Если вы хотите сравнить результаты разных моделей, создайте отдельные выравнивания для каждой.
