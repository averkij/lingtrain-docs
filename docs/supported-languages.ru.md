# Поддерживаемые языки {#supported-languages}

Lingtrain Aligner поддерживает широкий спектр языков для выравнивания текстов. На этой странице представлен полный перечень явно поддерживаемых языков, описание режима «General» для дополнительных языков и рекомендации по работе с различными письменностями.

### Подробности о разбиении на предложения {#splitting-details}

Lingtrain применяет различные стратегии разбиения в зависимости от языка:

**Разбиение на основе razdel** (русский, украинский, белорусский): библиотека `razdel` — это алгоритм разбиения на предложения, разработанный специально для текстов на кириллице. Он корректно обрабатывает кириллические сокращения, кавычки и оформление диалогов (тире), характерное для русской литературы.

**Разбиение на основе регулярных выражений** (китайский, японский): эти языки используют полноширинные знаки пунктуации для обозначения конца предложения. Китайский разбивается по `。`, `！` и `？`. Японский использует аналогичные шаблоны с учётом японских кавычек и пунктуации.

**Стандартный токенизатор** (все остальные языки): универсальный токенизатор предложений, разбивающий по точке, восклицательному и вопросительному знакам, с эвристиками для сокращений, десятичных чисел и титулов (Mr., Dr. и т. п.). Хорошо работает для большинства языков с латиницей и кириллицей.

## Режим General {#general-mode}

Для языков, не входящих в явный список, выберите **«General»** (`xx`). В этом режиме для разбиения на предложения используется стандартный токенизатор, а межъязыковое сопоставление полностью опирается на модель эмбеддингов.

Модели эмбеддингов, используемые в Lingtrain (LaBSE, SONAR), поддерживают значительно более широкий набор языков. Вот примеры дополнительных языков, доступных через режим General:

## Языковой охват по моделям {#model-coverage}

Разные модели эмбеддингов, доступные в Lingtrain, имеют разный языковой охват:

| Модель | Примерное число языков | Лучше всего для |
|--------|----------------------|-----------------|
| distiluse-base-multilingual-cased-v2 | 50+ | Основные европейские и азиатские языки |
| LaBSE | 109 | Широкий многоязычный охват, малоресурсные языки |
| XLM-R-100langs | 100 | Широкий кросс-языковой охват |
| rubert-tiny2 | Русский + ряд других | Задачи, ориентированные на русский (быстрый инференс) |
| SONAR | 200+ | Самый широкий охват, крайне малоресурсные языки |

Доступные модели для конкретной языковой пары отображаются в селекторе моделей при создании или настройке выравнивания.

## Советы для конкретных письменностей {#script-tips}

### CJK-языки (китайский, японский, корейский) {#cjk-tips}

- **Китайский**: убедитесь, что в тексте используется стандартная китайская пунктуация (`。`, `！`, `？`, `，`). Смешение полноширинной и обычной пунктуации может вызвать проблемы с разбиением. Традиционный и упрощённый китайский обрабатываются одинаково — модель эмбеддингов справляется с обоими.

- **Японский**: алгоритм разбиения ожидает японскую пунктуацию конца предложения. Для текстов, в которых японский перемежается с английскими цитатами, алгоритм обработает смешанный контент, но длинные вставки на английском могут разбиваться неоптимально.

- **Корейский**: в корейском используется сочетание конечных частиц и пунктуации. Стандартный токенизатор справляется с корейским достаточно хорошо, но для лучших результатов используйте стандартную корейскую пунктуацию (`.`, `!`, `?`).

### Арабское письмо {#arabic-tips}

- Арабский, персидский, урду и другие языки с арабским письмом поддерживаются через режим General
- Отображение справа налево обрабатывается браузером; специальная настройка не требуется
- Убедитесь, что в тексте используются стандартные арабские знаки пунктуации (`.`, `؟`, `!`)
- Для арабского модель эмбеддингов работает как со стандартным арабским, так и с распространёнными диалектами

### Кириллица {#cyrillic-tips}

- Русский, украинский и белорусский имеют выделенные алгоритмы разбиения, обрабатывающие специфические для кириллицы сокращения и диалоговое оформление
- Другие языки с кириллическим письмом (сербский, болгарский, македонский, казахский, киргизский, монгольский и др.) должны использовать режим General
- Алгоритм `razdel` для русского/украинского/белорусского корректно обрабатывает оформление диалогов с тире (— Привет! — сказал он.), характерное для русской литературы

### Деванагари и индийские письменности {#indic-tips}

- Хинди, маратхи, непальский (деванагари), бенгальский, гуджарати, каннада, малаялам, тамильский, телугу — все поддерживаются через режим General
- Стандартный токенизатор работает с индийским знаком конца предложения (пурна вирам `।`), если он присутствует
- Для лучших результатов обеспечьте единообразную пунктуацию по всему тексту

## Подстрочник для малоресурсных языков {#proxy-for-low-resource}

Для языков с ограниченным представлением в модели эмбеддингов качество выравнивания можно значительно улучшить с помощью [подстрочника](proxy.ru.md). Рекомендуемый подход:

1. Загрузите текст на малоресурсном языке
2. Скачайте разбитый текст
3. Переведите его машинным переводом на хорошо поддерживаемый язык (английский или русский)
4. Загрузите перевод как прокси-документ

Это особенно рекомендуется для:
- Башкирского, чувашского и других тюркских миноритарных языков
- Языков с числом носителей менее 10 миллионов
- Языковых пар, в которых ни один из языков не является английским, русским, китайским или другим крупным языком
